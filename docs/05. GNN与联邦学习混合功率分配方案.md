# GNN + 联邦学习混合功率分配方案

> **对应国自然方向**：方向(4) 基于联邦学习的分布式预编码
> **目标**：通过 GNN 实现本地分布式推理（消除同步等待），通过联邦学习实现 AP 间协作优化（弥补信息损失）
> **文档版本**：v1.0 | 2026-04-26

---

## 1. 问题建模

### 1.1 功率分配问题形式化

在 Cell-Free 分布式 MIMO 下行链路中，功率分配的目标是最大化系统和速率（ESR），约束为总功率上限 $P_t$：

$$
\max _{\boldsymbol{\rho}} \quad \sum_{k=1}^{K} \mathrm{ESR}_k(\boldsymbol{\rho}) \quad \text{s.t.} \quad \sum_{l=1}^{L} \rho_{l,k} = 1, \quad \rho_{l,k} \geq 0
$$

其中 $\rho_{l,k}$ 是分配给用户 $k$ 从 AP $l$ 发信号的功率比例，$\sum_l \rho_{l,k} = 1$ 表示该用户的功率在各 AP 间归一化。

**核心矛盾**：现有 WMMSE/PSO 等方法需要集中式优化——CPU 必须收集所有 AP 的完整 CSI 才能计算，AP 间存在大量信息交换，带来同步时延。

### 1.2 同步时延的根源

```
集中式 WMMSE 的同步代价：
┌──────────────────────────────────────────────────────────────┐
│  AP_1 ──CSI──→ CPU ←──CSI── AP_2 ←──CSI── ... ←──CSI── AP_L │
│                ↓                                            │
│           集中计算 ρ_{l,k} 矩阵                                │
│                ↓                                             │
│  AP_1 ←─ρ── CPU ──ρ→ AP_2 ←─ρ→ ... ←─ρ── AP_L             │
└──────────────────────────────────────────────────────────────┘
同步轮数: O(L)    CSI传输量: O(L×K)    ρ下发量: O(L×K)
```

### 1.3 解决思路：GNN 本地推理 + 联邦学习协作

```
GNN+FL 混合方案的同步代价：
┌──────────────────────────────────────────────────────────────┐
│  每个AP: 本地 CSI → GNN(本地推理) → ρ_{l,*} (无需通信!)       │
│                ↓ (仅模型参数)                                  │
│           UCDU: 聚合 GNN 参数 → 全局模型 → 分发               │
│                 (仅 K 维向量，不是 L×K CSI 矩阵)               │
└──────────────────────────────────────────────────────────────┘
同步轮数: O(通信轮数, 通常 < 20)   传输量: O(K) 模型参数
```


### 2.2 GNN + FL 协作流程

```
第1轮: 各 AP 用本地 CSI 输入 GNN → 得到局部 ρ^(1)
         ↓
第2轮: 各 AP 将 GNN 参数梯度上传到 UCDU
         ↓
       UCDU 加权聚合 → 全局 GNN 参数
         ↓
       UCDU 将全局参数下发给所有 AP
         ↓
第3轮: 各 AP 用全局 GNN 参数 + 本地 CSI → 更新 ρ^(2)
         ↓
     ... 重复直到收敛（或固定 10~20 轮）
         ↓
     最终 ρ* 收敛（接近集中式 WMMSE 性能）
```

## 3. GNN 模块设计

### 3.1 图建模

将 Cell-Free 系统建模为**用户-接入点（UE-AP）二部图**：

- **节点**：$L$ 个 AP 节点 + $K$ 个 UE 节点
- **边**：当 $d_{l,k} = 1$（AP $l$ 服务 UE $k$）时存在边 $(l,k)$
- **边的方向**：AP → UE（功率从 AP 流向 UE）

```
    AP_1 ────ρ_{1,1}──→ UE_1
     │                      ↑
     │ ρ_{1,K}              │ d_{L,1}
     ↓                      ↓
    AP_L ────ρ_{L,1}──→ UE_K ←── d_{L,K}
```

### 3.2 节点特征设计

每个节点的输入特征（来自 MATLAB 仿真数据）：

| 特征维度 | 来源变量 | 说明 |
|---------|---------|------|
| AP 节点 | `gainOverNoise(l,k)` | AP $l$ 到所有 UE 的 大尺度衰落（L×K 矩阵flatten） |
| AP 节点 | `Dl(:,l)` | AP $l$ 的接入模式向量（K 维，0/1） |
| AP 节点 | `B`（标量广播） | 广播功率因子（可从总功率 $P_t$ 得到） |
| UE 节点 | `gainOverNoise(:,k)` | 所有 AP 到 UE $k$ 的 大尺度衰落 |
| UE 节点 | `sigma_e` | 信道估计误差方差 |

**简化版本（优先实现）**：每个 AP 节点的特征为 $\mathbf{f}_l = [\sqrt{g_{l,1}}, \sqrt{g_{l,2}}, \ldots, \sqrt{g_{l,K}}]$（K 维），每个 UE 节点特征为 $\mathbf{f}_k = [\sqrt{g_{1,k}}, \sqrt{g_{2,k}}, \ldots, \sqrt{g_{L,k}}, \sigma_e]$（L+1 维）。

### 3.3 边特征设计

| 边特征 | 来源 | 维度 |
|-------|------|-----|
| 大尺度衰落 | $\sqrt{g_{l,k}}$ | 标量 |
| 接入指示 | $d_{l,k}$ | 标量（0/1） |
| 归一化距离 | $d_{l,k} / d_{\max}$ | 标量 |

### 3.4 网络架构

采用 **GraphSAGE** 或 **GAT（Graph Attention Network）** 架构：

```python
import torch.nn as nn
import torch_geometric.nn as pyg_nn

class PowerGNN(nn.Module):
    def __init__(self, in_dim_ap, in_dim_ue, hidden_dim=64, out_dim=1, num_heads=4):
        super().__init__()
        # AP 更新 MLP
        self.ap_mlp = nn.Sequential(
            nn.Linear(in_dim_ap, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim)
        )
        # UE 更新 MLP
        self.ue_mlp = nn.Sequential(
            nn.Linear(in_dim_ue, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim)
        )
        # AP → UE 消息（边卷积）
        self.ap_to_ue_conv = pyg_nn.GATConv(hidden_dim, hidden_dim // num_heads, heads=num_heads, concat=True)
        # UE → AP 消息（边卷积）
        self.ue_to_ap_conv = pyg_nn.GATConv(hidden_dim, hidden_dim // num_heads, heads=num_heads, concat=True)
        # 最终输出层：每个 AP 节点输出 K 个 ρ 值
        self.output = nn.Linear(hidden_dim, hidden_dim)
        self.rho_head = nn.Linear(hidden_dim, 1)  # 输出该 AP 到其所有关联 UE 的功率

    def forward(self, x_ap, x_ue, edge_index_ap_ue, edge_index_ue_ap):
        # x_ap: [L, in_dim_ap], x_ue: [K, in_dim_ue]
        # edge_index_ap_ue: [2, num_edges], edge_index_ue_ap: [2, num_edges]

        # AP 节点初始化
        h_ap = self.ap_mlp(x_ap)  # [L, hidden_dim]
        # UE 节点初始化
        h_ue = self.ue_mlp(x_ue)  # [K, hidden_dim]

        # 消息传递：2~3 层 GraphSAGE/GAT
        for _ in range(2):
            # AP → UE：UE 聚合来自 AP 的信息
            h_ue_new = self.ap_to_ue_conv(h_ap, edge_index_ap_ue)  # [K, hidden_dim]
            h_ue_new = torch.relu(h_ue_new)
            # UE → AP：AP 聚合来自 UE 的信息
            h_ap_new = self.ue_to_ap_conv(h_ue, edge_index_ue_ap)  # [L, hidden_dim]
            h_ap_new = torch.relu(h_ap_new)
            h_ap, h_ue = h_ap_new, h_ue_new

        # 最终输出：每个 AP 输出到每个 UE 的功率比例
        rho_logits = self.rho_head(self.output(h_ap))  # [L, 1]
        return rho_logits  # 配合 softmax 得到归一化 ρ
```

### 3.5 输出层设计

- **输出**：对每个 AP $l$，输出 $\tilde{\rho}_{l,k}$（未归一化），经过 Softmax 归一化：
  $$\rho_{l,k} = \frac{\exp(\tilde{\rho}_{l,k})}{\sum_{j: d_{l,j}=1} \exp(\tilde{\rho}_{l,j})}$$
- **损失函数**：
  - **主损失**：负和速率 $- \sum_k \mathrm{ESR}_k$（来自 MATLAB 仿真数据中的标签）
  - **辅助损失**：与 WMMSE/PSO 结果的 MSE 回归损失（可选，用于稳定训练）

## 4. 联邦学习模块设计

### 4.1 联邦学习框架

采用 **FedAvg（联邦平均）** 框架：

```
Algorithm: FedAvg for Power Allocation

初始化:
  - 全局 GNN 模型参数 θ_global
  - 每轮参与 AP 集合 S_t ⊆ {1,...,L}
  - 每轮参与率 C = |S_t| / L

每轮 t 执行:
  1. UCDU 将 θ_global 下发给参与 AP 集合 S_t
  2. 每个 AP l ∈ S_t 执行本地训练:
       θ_l^(t+1) = LocalUpdate(θ_global, local_data_l)
  3. UCDU 聚合: θ_global^(t+1) = Σ_{l∈S_t} (n_l / n_total) · θ_l^(t+1)
     其中 n_l 为 AP l 的本地数据集大小

LocalUpdate (每个 AP 本地执行):
  - 用本地 CSI 数据集 D_l
  - 执行 E 轮本地 SGD 更新
  - 返回更新后的模型参数
```

### 4.2 参与策略

- **全参与（Full Participation）**：每轮所有 L 个 AP 都参与 → 最稳定但时延高
- **随机采样（Partial Participation）**：每轮随机选择 $C \times L$ 个 AP → 降低通信开销
- ****推荐**：随机采样 $20\% \sim 50\%$ AP 参与，平衡性能和时延

### 4.3 聚合策略

采用**加权联邦平均**，权重与各 AP 的本地数据量成正比：

$$
\theta_{\text{global}} = \sum_{l \in \mathcal{S}_t} \frac{n_l}{n_{\mathcal{S}_t}} \theta_l
$$

其中 $n_l$ 是 AP $l$ 的本地训练样本数，$n_{\mathcal{S}_t}$ 是参与集合的总样本数。

### 4.4 通信轮数与收敛

- **目标**：10~20 轮联邦通信内收敛
- **收敛判定**：连续 3 轮全局 ESR 增益 < 0.5%
- **与 OTA（空口聚合）的结合**：所有 AP 同时发送模型参数到 UCDU，利用信道叠加进一步降低通信开销（后续可选扩展）

### 5.2 推理接口（Python → MATLAB）

新增 `computeRhoGNN.m`，放在 `power_allocation/` 目录下：

```matlab
function [rho] = computeRhoGNN(Hhat, D, gainOverNoise, Pt, modelPath)
% computeRhoGNN - 基于训练好的 GNN 模型进行功率分配
% 输入:
%   Hhat          - 信道估计 (L·N × nbrOfRealizations × K)
%   D             - 接入模式矩阵 (L × K, 0/1)
%   gainOverNoise - 大尺度衰落/噪声比 (L × K)
%   Pt            - 总发射功率（瓦特）
%   modelPath     - PyTorch GNN 模型路径 (.pt 文件)
% 输出:
%   rho           - 功率分配矩阵 (L × K)
%                    每列归一化和为 Pt（即 Σ_l ρ_{l,k} = Pt 不对，
%                    实际是每行归一化 Σ_k ρ_{l,k} = Pt）

    % 1. 构建节点特征
    L = size(D, 1);
    K = size(D, 2);

    sqrt_g = sqrt(gainOverNoise);  % [L × K]

    % AP 节点特征: 每行是一个 AP 到所有 UE 的衰落系数
    x_ap = sqrt_g';  % [K × L] → 转置后每列是一个 AP 的特征向量
    % 但 GNN 期望 [num_nodes, feature_dim]，需要 flatten 处理
    x_ap = sqrt_g;   % [L × K]

    % UE 节点特征
    x_ue = sqrt_g';  % [K × L]
    ue_extra = sigma_e * ones(K, 1);  % 信道估计误差
    x_ue = [x_ue, ue_extra];  % [K × (L+1)]

    % 2. 构建边索引
    [src_ap, dst_ue] = find(D == 1);  % AP → UE 的边
    edge_ap_ue = [src_ap'; dst_ue'];  % [2, num_edges]
    edge_ue_ap = [dst_ue'; src_ap'];  % [2, num_edges]

    % 3. 调用 Python GNN 推理（使用 system Python）
    % 构造 Python 可读的 CSV/JSON 临时文件
    tmp_feat_ap = fullfile(tempdir, 'gnn_x_ap.csv');
    tmp_feat_ue = fullfile(tempdir, 'gnn_x_ue.csv');
    tmp_edge = fullfile(tempdir, 'gnn_edge.csv');
    csvwrite(tmp_feat_ap, x_ap);
    csvwrite(tmp_feat_ue, x_ue);
    dlmwrite(tmp_edge, edge_ap_ue, 'delimiter', ',');

    pyCmd = sprintf(['python -c "import torch; ', ...
        'model = torch.load(''%s'', map_location=''cpu''); ', ...
        'model.eval(); ', ...
        '... (加载数据，执行推理，输出 rho)"], modelPath);
    [status, rho_flat] = system(pyCmd);
    rho = reshape(rho_flat, L, K);

    % 4. 归一化（每行和为 Pt）
    rowSum = sum(rho, 2);
    rho = rho ./ rowSum .* Pt;

    % 清理临时文件
    delete(tmp_feat_ap, tmp_feat_ue, tmp_edge);
end
```

## 6. 实施步骤

### Phase 1: 数据集生成（1~2 天）

**目标**：生成高质量的 (CSI, 最优 ρ) 训练数据集

1. 修改/新增 `exportTrainingData.m`：
   - 运行 `Combined_Downlink_Sim.m`，每个 SNR 点生成 500~1000 个快照
   - 用 WMMSE 作为"教师模型"生成最优 ρ 标签
   - 额外生成 DCC 模式下的数据（不同 AP-UE 关联矩阵）
   - 导出 `sqrt(gainOverNoise)`、`D`、`sigma_e`、`rho_WMMSE`

2. 数据增强策略：
   - **拓扑扰动**：随机丢弃 10%~30% 的 AP-UE 连接，模拟稀疏接入
   - **噪声扰动**：对 `sigma_e` 添加 $\pm 20\%$ 随机扰动
   - **SNR 插值**：训练 SNR 范围 [-10, 30] dB，间隔 5 dB

**数据量估算**：
- L=100, K=20 → 每个快照约 100×20 = 2000 个数据点
- 4 SNR × 500 快照 = 2000 个快照
- 总计约 2000 × 2000 = 400 万条边级别的训练样本

### Phase 2: GNN 模型训练（2~3 天）

**目标**：训练一个泛化能力强的 GNN 功率分配器

1. **环境准备**：
   ```bash
   # Python 3.9+, PyTorch 2.0+, PyTorch Geometric
   pip install torch torchvision torch-geometric scipy pandas
   ```

2. **网络训练** (`train_gnn.py`)：
   ```python
   # 训练主循环
   for epoch in range(num_epochs):
       for batch in dataloader:
           # 前向传播
           rho_pred = gnn(x_ap_batch, x_ue_batch, edge_index_batch)
   ```

### Phase 4: 同步性能评估（1~2 天）

**目标**：量化 GNN+FL 方案相比集中式方法的同步时延收益

1. **对比基线**：
   - 基线 1：集中式 WMMSE（`computeRhoWMMSE.m`）
   - 基线 2：分布式大尺度衰落比例（`computeRhoDist.m`）
   - 基线 3：GNN 本地推理（无 FL，单 AP 独立使用）

2. **评估指标**：
   | 指标 | 定义 | 测量方法 |
   |------|------|---------|
   | **ESR 损失** | $1 - \frac{\mathrm{ESR}_{\text{GNN+FL}}}{\mathrm{ESR}_{\text{WMMSE}}}$ | 对比图 |
   | **同步时延降低** | $\frac{T_{\text{集中式}} - T_{\text{GNN+FL}}}{T_{\text{集中式}}}$ | 计时 |
   | **通信开销降低** | $\frac{D_{\text{集中式}} - D_{\text{GNN+FL}}}{D_{\text{集中式}}}$ | 传输数据量 |

## 8. 技术风险与缓解

| 风险 | 影响 | 缓解策略 |
|------|------|---------|
| GNN 训练不收敛 | ESR 损失过大 | 1. 用 WMMSE 结果做监督学习（而非纯 ESR） 2. 预训练 + 微调策略 |
| 联邦学习通信开销过高 | 失去同步优势 | 采用参数压缩（量化到 8-bit）+ 稀疏化（仅传 10% 最重要参数） |
| 拓扑变化时 GNN 泛化差 | 换拓扑需重训 | 拓扑扰动数据增强；图结构作为边特征而非节点位置编码 |
| MATLAB ↔ Python 接口延迟 | 在线推理速度 | 用 MATLAB Engine API for Python（直接调用，无需文件 I/O） |
| L=100 时 GNN 规模过大 | 训练慢 | 图采样（GraphSAINT）或只保留每个 AP 的 Top-20 邻居 UE |
