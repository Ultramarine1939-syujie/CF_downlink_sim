# 项目运行指南

> 最后更新：2026-05-12 | 当前实现：MATLAB 仿真 + GNN 功率分配

本文档说明当前项目的完整运行流程：MATLAB 生成信道与标签、Python 训练 GNN、MATLAB 调用 GNN 推理并绘制结果。

## 快速运行

在项目根目录执行：

```powershell
run_test.bat 5 /y
```

该批处理会完成环境检查、数据导出、GNN 快速训练、Python/MATLAB 推理冒烟测试和完整 MATLAB 仿真。参数 `5` 表示每个 SNR 点导出 5 个快照；`/y` 表示跳过交互确认。

如果只想运行主仿真：

```powershell
matlab -batch "cd('C:\Users\Admin\Documents\个人资料\CF_downlink_sim'); run"
```

## 手动流程

### Step 1：安装 Python 依赖

```powershell
cd python
pip install -r requirements.txt
cd ..
```

### Step 2：导出训练数据

```powershell
matlab -batch "cd('C:\Users\Admin\Documents\个人资料\CF_downlink_sim'); addpath(genpath(pwd)); exportTrainingData(fullfile(pwd,'data','gnn_training'), 50)"
```

导出的 `.mat` 文件包含：

| 字段 | 说明 |
|---|---|
| `D` | AP-UE 服务矩阵 |
| `gainOverNoisedB` | 大尺度信道增益 |
| `rho_WMMSE` | WMMSE 功率分配标签 |
| `sigma_e` | CSI 误差强度 |

### Step 3：训练 GNN

```powershell
cd python
python train_gnn.py --data "../data/gnn_training/*.mat" --epochs 300
python train_gnn_local.py --data "../data/gnn_training/*.mat" --epochs 120
cd ..
```

快速验证可把 `--epochs` 改成较小值，例如：

```powershell
cd python
python train_gnn.py --data "../data/gnn_training/*.mat" --epochs 10 --batch_size 64
python train_gnn_local.py --data "../data/gnn_training/*.mat" --epochs 10
cd ..
```

训练产物位于 `models/`：

| 文件 | 说明 |
|---|---|
| `best_gat_gnn_power.pt` | MATLAB 仿真默认加载的 GNN 模型 |
| `best_local_gnn_power.pt` | Local-GNN 分布式推理模型 |
| `final_gat_gnn_power.pt` | 最后一轮模型 |
| `training_log.csv` | 训练日志 |

### Step 4：运行完整仿真

```powershell
matlab -batch "cd('C:\Users\Admin\Documents\个人资料\CF_downlink_sim'); run"
```

`run.m` 会自动加入项目路径并执行 `main/Combined_Downlink_Sim.m`。如果 `models/best_gat_gnn_power.pt` 不存在，GNN 功率分配会回退到 EPA，便于流程不中断。

## 算法组合

当前主仿真遍历 **64 种组合 = 8 PA x 4 PC x 2 Mode**。

功率分配（PA）：

| 方法 | 代码名 | 说明 |
|---|---|---|
| Baseline | `baseline` | 距离/大尺度增益基线 |
| Random | `random` | 随机功率后按 AP 归一化 |
| EPA | `EPA` | 等功率分配 |
| D-WMMSE | `DWMMSE` | 固定轮次、有限消息交换的分布式 WMMSE 近似 |
| Local-GNN | `LocalGNN` | 每个 AP 独立使用本地 AP-UE 行推理 |
| WMMSE | `WMMSE` | 迭代优化标签源与性能上界参考 |
| PSO | `PSO` | 粒子群优化 |
| GNN | `GNN` | 全图 GAT-GNN 低时延参考 |

预编码（PC）：

| 方法 | 说明 |
|---|---|
| MR | 最大比传输 |
| L-MMSE | 分布式线性 MMSE |
| R-MMSE | 鲁棒 MMSE |
| L-MMSE-G | 全局协方差 MMSE 基准 |

当前结果展示会区分两类方法：

| 类别 | 说明 |
|---|---|
| Distributed candidates | PC 和 PA 均满足分布式信息约束，参与主排名 |
| Centralized/offline references | 用于性能上界、教师标签或消融参考，不参与分布式主排名 |

注意：`GNN` 仍是一次全图推理，因此被标记为 low-latency centralized reference；`LocalGNN` 是当前严格分布式 GNN 最小版本，会进入分布式主排名。若 `models/best_local_gnn_power.pt` 缺失，Local-GNN 会自动回退到 EPA，保证仿真流程不中断。

接入模式：

| 模式 | 说明 |
|---|---|
| All | 所有 AP 服务所有 UE |
| DCC | 动态协作簇 |

## 缓存说明

`main/Combined_Downlink_Sim.m` 默认启用缓存。缓存指纹会记录系统参数和 GNN 模型文件信息；修改模型、算法表或关键参数后会自动触发相应重算。

常用设置：

| 目标 | 设置 |
|---|---|
| 快速验证 | 减小 `numScenarios`、`nbrOfRealizations` 或 SNR 点数 |
| 只跑传统算法 | `runStage = 1` |
| 只重跑 GNN | `runStage = 2` |
| 跑完整流程 | `runStage = 3` |
| 强制重算 | 删除 `main/SimulationData/cache_*.mat` 或设置 `useCache = false` |
| 关闭同步消融图 | `enableSyncAblation = false` |

同步时延模型位于 `main/Combined_Downlink_Sim.m` 用户配置区：

| 参数 | 含义 |
|---|---|
| `syncAblation.fronthaulMbps` | 控制/前传链路速率，用于把同步载荷折算为传输时延 |
| `syncAblation.syncRttMs` | 单轮 AP/CPU 同步控制往返时延 |
| `syncAblation.dccPayloadRatio` | DCC 相对 All 模式的同步载荷比例近似 |
| `syncAblation.wmmseRounds` | WMMSE 功率分配的同步轮次 |
| `syncAblation.psoRounds` | PSO 功率分配的全局最优同步轮次 |

## 常见问题

### GNN 结果仍然很差？

优先确认三件事：

1. `models/best_gat_gnn_power.pt` 是否是新训练模型。
2. `main/Combined_Downlink_Sim.m` 的缓存版本是否已更新，或缓存是否已删除。
3. 绘图是否使用当前 `algoTable` 精确匹配 PA/PC/Mode，而不是旧的行号假设。

### 可以只靠 `.bat` 完成测试吗？

可以。当前 `run_test.bat 5 /y` 已覆盖完整测试链路，包括 Python 编译检查、数据导出、训练、推理冒烟测试和 MATLAB 主仿真。

### MATLAB 调 Python 失败？

在 MATLAB 中检查：

```matlab
pyenv
py.importlib.import_module('torch')
```

如 Python 路径不正确，使用：

```matlab
pyenv('Version', '你的Python路径\python.exe')
```
