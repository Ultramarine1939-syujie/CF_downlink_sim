## Step 5a：缓存与分阶段运行

### 概述

v3.0 引入了**结果缓存**和**分阶段控制**，解决 WMMSE 和 GNN 推理耗时过长的问题。

核心机制：
- 第一次运行时自动将所有中间结果（信道数据、预编码向量、功率分配、ESR）缓存到 `SimulationData/` 目录
- 后续运行时检测参数是否变化，未变化则直接复用缓存结果
- 更换了 GNN 模型文件后，仅重跑 GNN 推理，传统方法结果直接复用

### 配置参数

在 `Combined_Downlink_Sim.m` 用户配置区（约第 33-38 行）：

```matlab
runStage = 3;     % 1=仅传统方法, 2=仅GNN(需stage1缓存), 3=全部(默认)
useCache = true;   % 缓存总开关; false 则每次全部重算(等同v2.0)
```

### 运行场景

| 场景 | 设置 | 行为 |
|------|------|------|
| 首次运行 | `runStage=3` (默认) | 全部计算 + 写缓存 |
| 参数未变重跑 | `runStage=3` | 跳过全部计算，直接出图 |
| 只跑传统方法 | `runStage=1` | WMMSE/PSO/Baseline/EPA/Random |
| 只跑GNN | `runStage=2` | 需先有 stage1 缓存，仅重跑 GNN 推理 |
| 换了 .pt 模型 | `runStage=2` | 传统结果复用，GNN 部分重算 |
| 调了参数 (如sigma_e) | `runStage=3` | 自动检测参数变化，全部重算 |
| 强制无缓存 | `useCache=false` | 等价 v2.0，每次从头计算 |

### 缓存文件

缓存存储在 `main/SimulationData/` 目录下，分为两级：

| 缓存文件 | 内容 | 共享范围 |
|---------|------|---------|
| `cache_scenario_s{n}.mat` | 信道 Hhat/H/C、D 矩阵、拓扑 | 一个场景的所有 SNR 点 |
| `cache_snr_s{n}_sn{SNR}.mat` | 预编码 V、所有 rho、ESR 结果、模型时间戳 | 一个 (场景, SNR) 组合 |

> 不会自动删除缓存文件。如需全部重算，手动删除 `SimulationData/` 下的 `cache_*.mat` 文件即可。

### 缓存有效性判定

- **参数指纹**：MD5 哈希覆盖 L, K, N, sigma_e, SNR_dB, nbrOfRealizations, ASD, tau_c, tau_p, nIter
- **模型变更检测**：比对 `.pt` 文件的修改时间（~0.1秒容差）
- **级联失效**：stage1（传统方法）失效时，stage2（GNN）也会强制重算

### 控制台输出示例

缓存命中时输出：
```
  [CACHE HIT] Scenario 1 loaded
  |   SNR  1/ 6 (  5 dB)  cache: stage1=true stage2=true
  |   |  [Stage 1 loaded from cache]
  |   |  [Stage 2 loaded from cache]
```

部分失效时输出：
```
  [CACHE MISS] Scenario 1 param mismatch, regenerating
  |   SNR  1/ 6 (  5 dB)  cache: stage1=false stage2=false
  |   |  [Stage 1] Precoding... PSO... WMMSE... done
  |   |  [GNN] Inference (All)... done
```

## 修改仿真参数

`Combined_Downlink_Sim.m` 是脚本（不是函数），所有参数通过直接编辑文件修改。关键参数区域位于文件开头（约第 8-38 行）：

```matlab
% ========== 用户配置区 ==========
L   = 100;          % AP 数量
K   = 20;           % UE 数量
N   = 1;            % 每 AP 天线数
tau_c = 200;        % 相干块长度
tau_p = 10;         % 导频长度
SNR_dB = 5:5:30;    % SNR 扫描范围 [dB]
nbrOfRealizations = 200;  % 信道实现次数
% ================================

% ========== 缓存 & 阶段控制 (v3.0) ==========
runStage = 3;     % 1=传统方法, 2=GNN, 3=全部
useCache = true;   % 缓存总开关
% =============================================
```

> 修改任何仿真参数（L/K/sigma_e/SNR等）后，缓存会自动失效并重新计算，无需手动清理。

## 算法组合说明

仿真自动遍历以下组合（共 56 种 = 7 PA x 4 PC x 2 Mode）：

### 功率分配（PA）

| 名称 | 关键字 | 阶段 | 说明 |
|------|--------|------|------|
| Baseline | `baseline` | 传统 | 基于大尺度衰落距离比 |
| 等功率 | `EPA` | 传统 | 每个服务的 UE 平分功率 |
| WMMSE | `WMMSE` | 传统 | 加权最小均方误差迭代（最慢） |
| PSO | `PSO` | 传统 | 粒子群优化 |
| Random | `random` | 传统 | 随机分配后归一化 |
| GNN | `GNN` | GNN | GAT-GNN 本地推理 |
| GNN+FL | `GNNFL` | GNN | GNN + FedAvg 联邦微调 |

### 预编码（PC）

| 名称 | 关键字 | 说明 |
|------|--------|------|
| MR | `MR` | 最大比传输 |
| L-MMSE | `LMMSE` | 线性最小均方误差（分布式） |
| R-MMSE | `RMMSE` | 鲁棒 MMSE |
| L-MMSE-G | `LMMSE_G` | 全局协方差（集中式，消融用） |

### 接入模式（AP Mode）

| 名称 | 说明 |
|------|------|
| All | 所有 AP 服务所有 UE |
| DCC | 基于大尺度衰落的动态 AP-UE 关联 |

### GNN 消融模型（可选，需模型文件）

| 名称 | 说明 | 模型文件 |
|------|------|---------|
| GNN-MLP | MLP 替代 GAT（消融维度1+2） | `models/ablation/mlp_only/best_model.pt` |
| GNN-GN | 全局归一化替代样本内归一化（消融维度4） | `models/ablation/global_norm/best_model.pt` |

> 消融模型仅计算 rho 值并缓存，不参与 parfor SE 评估。如需完整 ESR 对比，将其注册到 PA 注册表中。

## 常见问题

### Q: 运行 run.m 时 GNN 算法没有结果？

检查 `models/` 目录下是否有 `best_gat_gnn_power.pt` 和 `gnn_power_fedavg.pt`。如果没有，需先完成 Step 1-3 训练模型。模型文件缺失时 GNN 会自动回退到 EPA。

### Q: MATLAB 调用 Python 失败？

在 MATLAB 中配置 Python 解释器路径：

```matlab
% 检查当前 Python 配置
pyenv

% 设置 Python 路径（按需修改）
pyenv('Version', 'C:\Users\Admin\AppData\Local\Programs\Python\Python310\python.exe');
```

### Q: 如何只跑部分算法组合？

编辑 `Combined_Downlink_Sim.m` 中约第 55-91 行的算法注册表，注释掉不需要的算法即可。

### Q: WMMSE 计算很慢？

v3.0 已引入缓存机制，WMMSE 结果会自动缓存到 `SimulationData/cache_snr_*.mat`。重复运行时直接复用，无需重算。

如果首次运行仍然觉得慢，可以：
- 减少 `nbrOfRealizations`（如改为 50）做快速验证
- 减少 SNR 点数（如改为 `[10, 20, 30]`）
- 使用 `runStage=1` 仅跑传统方法，`runStage=2` 仅跑 GNN（传统结果从缓存加载）

### Q: 如何清除缓存重新计算？

删除 `main/SimulationData/` 下的 `cache_*.mat` 文件，或设置 `useCache = false`。

### Q: 换了 GNN 模型后需要全部重跑吗？

不需要。将新的 `.pt` 文件替换到对应路径后，设置 `runStage=2`，脚本会自动检测模型文件变更，仅重跑 GNN 推理，传统方法结果从缓存复用。

### Q: 导出训练数据时报错？

确保已执行 `addpath(genpath(pwd))`，且当前目录在 `CF_downlink_sim/` 下。`exportTrainingData.m` 依赖 `power_allocation/` 下的多个函数。
