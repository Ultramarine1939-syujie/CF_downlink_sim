> **项目名称**：CF_downlink_sim — Cell-Free 大规模 MIMO 下行链路仿真系统
> **语言**：MATLAB + Python 混合架构
> **最后更新**：2026-05-10
> **版本**：v3.0（性能优化版）

## 一、系统总览

本项目是一个 **Cell-Free 分布式 MIMO 下行链路功率分配研究平台**，核心目标是通过 GNN（图神经网络）实现低延迟的分布式功率分配，用联邦学习（FL）在 AP 间协作优化，逼近集中式 WMMSE 的性能。

### 数据流全景


### 2.2 MATLAB 与 Python 的协作边界

| 环节 | 执行者 | 原因 |
|------|--------|------|
| 信道仿真 (蒙特卡洛) | MATLAB | 矩阵运算密集，parfor 加速 |
| 预编码计算 (MR/L-MMSE/R-MMSE) | MATLAB | 与信道模型紧耦合 |
| WMMSE 迭代 (标签生成) | MATLAB | 需要完整 CSI 矩阵 |
| 训练数据导出 (.mat) | MATLAB | 仿真引擎的下游产物 |
| GNN 训练 (GAT + 消息传递) | Python | PyG 的绝对优势区 |
| 联邦学习 (FedAvg) | Python | PyTorch 分布式生态 |
| GNN 在线推理 | MATLAB 调 Python | 仿真循环内集成，通过 `py.importlib` |
| 结果汇总与绘图 | MATLAB | 与仿真结果统一处理 |

### 2.3 关键接口变量

MATLAB 与 Python 之间通过 `.mat` 文件交换以下数据：

| 变量 | 维度 | 说明 |
|------|------|------|
| `sqrtGain` | L x K x N_snaps | 大尺度衰落开方（GNN 输入特征） |
| `D` | L x K x N_snaps | AP-UE 接入矩阵 0/1（图边定义） |
| `sigma_e` | 1 x 1 x N_snaps | 信道估计误差（GNN 输入特征） |
| `rho_WMMSE` | L x K x N_snaps | WMMSE 最优功率分配（GNN 训练标签） |

推理时 MATLAB 直接调用 Python：

```
MATLAB (Hhat, D, gainOverNoise) → py.importlib 加载 .pt → GNN forward → ρ_{l,k}
```

## 四、创新点

### 创新点 1：GNN 本地推理消除 AP 间 CSI 交换

**问题**：集中式 WMMSE 需要所有 AP 上传完整 CSI（O(L x K) 传输量），同步时延随 AP 数量线性增长。

**方案**：用 GNN 学习 WMMSE 的隐式映射关系，每个 AP 仅用本地 CSI 即可独立推理出功率分配。

```
传统 WMMSE:                    GNN 推理:
  AP_1 ──CSI──→ CPU             AP_1: CSI_local → GNN → ρ_{1,*}
  AP_2 ──CSI──→ CPU     vs      AP_2: CSI_local → GNN → ρ_{2,*}
  ...                           ...  (完全并行，零通信)
  AP_L ──CSI──→ CPU             AP_L: CSI_local → GNN → ρ_{L,*}
       ↓
  集中计算 → ρ 矩阵
  ↓
  AP_1 ←─ρ── CPU
  ...
```

**技术实现**：
- 图建模：UE-AP 二部图，D 矩阵定义边
- 网络架构：3 层 GATConv，多头注意力 (heads=4)，残差连接 + LayerNorm
- 输入特征：`sqrt(gainOverNoise)` + `D` + `sigma_e`
- 标签：WMMSE 迭代输出的 `rho` 矩阵（有监督学习）
- 模型规模：~22.4k 可训练参数，轻量级

### 创新点 2：联邦学习弥补分布式推理的信息损失

**问题**：纯本地 GNN 只见过部分信道状态，无法感知全局干扰格局，性能有天花板。

**方案**：通过 FedAvg 联邦聚合，让各 AP 协作优化 GNN 参数而不共享原始 CSI。

```
轮次 1:  UCDU 下发全局参数 θ_global
         ↓
轮次 2:  各 AP 用本地数据训练 → 得到 θ_l^(local)
         ↓
轮次 3:  上传 θ_l^(local) (模型参数, 非 CSI)
         UCDU 加权聚合: θ_global = Σ (n_l/n_total) · θ_l
         ↓
轮次 4:  下发更新后的 θ_global
         ... 重复 10~20 轮直到收敛
```

**关键设计**：
- 每轮随机采样 20%~50% AP 参与，降低通信开销
- 支持 Non-IID 数据划分（按 SNR / 模式 / SNR+模式）
- 热启动：从集中式训练的最优模型初始化全局模型
- 通信量：O(K) 模型参数 vs O(L x K) 原始 CSI

### 创新点 3：L-MMSE 预编码的分布式改造

**问题**：标准 L-MMSE 需要所有 AP 的协方差矩阵之和（全局信息），不符合分布式系统设计原则。

**方案**：消除 `cSum = sum(C, 4)` 的全局求和，每个 AP 仅使用本 AP 的协方差 `C(:,:,l,:)`。

```
改造前:  cSum(l) = Σ_{l'=1}^{L} C(:,:,l',:)    ← 需要所有 AP 的协方差
改造后:  c_l = C(:,:,l,:)                       ← 仅用本 AP 协方差
```

**效果**：L-MMSE 不再需要 AP 间 CSI 协方差信息交换，降低了同步时延和信令开销。SE 计算接口不变，与现有仿真框架完全兼容。

### 创新点 4：结构化仿真框架（v2.0）

**问题**：原版 30 种算法组合需要大量重复代码，扩展新算法时改动面大。

**方案**：引入 `algoTable` 结构数组作为算法注册表，统一管理所有组合。

```matlab
% 算法注册表 (部分)
algoTable.name     = {'MR_WMMSE_All', 'MR_WMMSE_DCC', ...};
algoTable.PA       = {'WMMSE',        'WMMSE',         ...};
algoTable.PC       = {'MR',           'MR',            ...};
algoTable.AP_MODE  = {'All',          'DCC',           ...};
```

**配套改进**：
- 实时最优追踪：仿真过程中显示 `New Best` 提示
- 排名表 + 相对增益百分比热力图
- 6 张聚焦最优发现的论文级图表

### 创新点 5：样本内归一化策略

**问题**：rho 值范围 [0, 36000] 但中位数约 0，全局归一化后标签集中在 -1 附近，GNN 学到常数预测即可。

**方案**：每个快照独立做 signed-log + 非零 min-max 缩放。

```
对于每个 snapshot s:
  零值 rho → y = -1 (固定, 表示无连接)
  非零 rho → signed_log(rho) → min-max 缩放到 [-1, +1]
```

**效果**：Val Corr 从 0.21 提升至 0.87（仅 15 epochs），训练收敛速度大幅改善。

## 五、GNN 模型架构

### 5.1 图结构

```
节点类型:
  AP 节点 × L=100  (特征: sqrtGain[l,:], D[l,:])
  UE 节点 × K=20   (特征: sqrtGain[:,k], sigma_e)

边类型:
  AP → UE (功率流向): D[l,k]=1 时存在
  共 ~1756 条边 (取决于 D 矩阵稀疏度)
```

### 5.2 网络结构

```
Input
  ├── AP 节点嵌入 (Linear: K → hidden_dim)
  └── UE 节点嵌入 (Linear: L+1 → hidden_dim)
        │
  Layer 1: GATConv(hidden_dim, hidden_dim/heads, heads=4) + Residual + LayerNorm
  Layer 2: GATConv(hidden_dim, hidden_dim/heads, heads=4) + Residual + LayerNorm
  Layer 3: GATConv(hidden_dim, hidden_dim/heads, heads=4) + Residual + LayerNorm
        │
  GlobalAttention Pooling (注意力加权池化)
        │
  AP Predictor Head (MLP + 残差输入拼接)
        │
  Output: tanh → [-1, +1] → 反归一化 → ρ_{l,k}
```

### 5.3 训练配置

| 超参数 | 值 |
|--------|-----|
| 隐藏维度 | 128 |
| 注意力头数 | 4 |
| GAT 层数 | 3 |
| Dropout | 0.1 |
| 学习率 (max) | 1e-3 (OneCycleLR) |
| 批大小 | 32 |
| Epochs | 300 |
| Early Stopping | patience=50 |
| 损失函数 | MSE + 相对误差 |

## 六、联邦学习框架

### 6.1 核心流程

```
初始化全局模型 θ_global

for round t = 1 to T:
    1. 随机采样 C × L 个 AP (C=0.2)
    2. 下发 θ_global 给选中 AP
    3. 各 AP 本地训练 E=1 轮 (AdamW, lr=5e-4)
    4. UCDU 聚合:
       θ_global = Σ (n_l / n_total) · θ_l
    5. 评估全局模型验证集指标
    6. 保存最优检查点
```

### 6.2 Non-IID 数据划分策略

| 策略 | `--split_by` | 说明 |
|------|-------------|------|
| 随机划分 | `random` | 每个 AP 随机分到数据样本 |
| 按 SNR 划分 | `snr` | 模拟不同位置的 AP 看到不同 SNR |
| 按接入模式划分 | `mode` | All/DCC 数据分开 |
| SNR + 模式 | `snr_mode` | 最接近真实非独立同分布 |

### 6.3 热启动

从集中式训练的最优模型初始化全局模型：

```bash
python fedavg.py --init_ckpt "../models/best_gat_gnn_power.pt" --rounds 30
```

## 七、数据增强与归一化

### 7.1 数据增强（exportTrainingData.m）

| 增强方式 | 参数 | 目的 |
|---------|------|------|
| 拓扑扰动 | 随机丢弃 10%~30% AP-UE 连接 | 提升拓扑泛化能力 |
| 噪声扰动 | sigma_e +/- 20% | 提升信道误差鲁棒性 |
| 双模式覆盖 | All + DCC | 同一信道两种接入模式 |

### 7.2 样本内归一化（dataset.py）

```
Step 1:  signed_log(x) = sign(x) × log₁(|x| + 1)
Step 2:  对非零值做 min-max 缩放到 [-1, +1]
Step 3:  零值保持 y = -1
```
