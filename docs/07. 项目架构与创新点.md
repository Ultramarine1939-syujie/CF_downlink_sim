# 项目架构与创新点

> 最后更新：2026-05-12 | 当前范围：GNN 功率分配，不包含协作聚合训练模块

## 一、系统总览

本项目是一个 Cell-Free Massive MIMO 下行链路功率分配与预编码仿真平台。当前核心目标是：用 GNN 学习 WMMSE 的功率分配规律，在 MATLAB 主仿真中以低延迟推理替代迭代式功率优化。

```mermaid
flowchart LR
    A["MATLAB 信道/拓扑生成"] --> B["WMMSE 标签生成"]
    B --> C[".mat 训练数据"]
    C --> D["Python GAT-GNN 训练"]
    D --> E["best_gat_gnn_power.pt"]
    E --> F["MATLAB 调 Python 推理"]
    F --> G["ESR 评估与绘图"]
```

## 二、目录分工

| 目录 | 职责 |
|---|---|
| `main/` | 主仿真入口、缓存、SNR 扫描和结果聚合 |
| `channel/` | Cell-Free 拓扑、信道估计和空间相关矩阵 |
| `precoding/` | MR、L-MMSE、R-MMSE、L-MMSE-G 预编码 |
| `power_allocation/` | Baseline、EPA、WMMSE、PSO、GNN 功率分配 |
| `python/` | GNN 数据集、训练、推理封装 |
| `visualization/` | ESR 曲线、PA 对比、消融图和结果打印 |
| `config/` | 默认参数与日志工具 |
| `docs/` | 项目说明文档 |

## 三、当前算法表

主仿真遍历 **48 种组合 = 6 PA x 4 PC x 2 Mode**。

功率分配：

| 方法 | 类型 | 作用 |
|---|---|---|
| Baseline | 传统 | 大尺度增益/距离基线 |
| Random | 传统 | 随机功率后归一化 |
| EPA | 传统 | 等功率分配 |
| WMMSE | 迭代优化 | GNN 标签源与性能参考 |
| PSO | 群智能优化 | 非凸搜索参考 |
| GNN | 深度学习 | 快速本地推理 |

预编码：

| 方法 | 作用 |
|---|---|
| MR | 低复杂度基线 |
| L-MMSE | 分布式线性 MMSE |
| R-MMSE | 考虑 CSI 误差的鲁棒 MMSE |
| L-MMSE-G | 全局协方差基准 |

接入模式：

| 模式 | 作用 |
|---|---|
| All | 全 AP 服务 |
| DCC | 动态协作簇 |

## 四、GNN 设计

当前 GNN 的训练和推理都使用同一套特征协议：

| 节点 | 特征 |
|---|---|
| AP | 对 UE 的 masked sqrt gain、SNR 归一化、CSI 误差、AP 度数、总增益 |
| UE | 来自 AP 的 masked sqrt gain、SNR 归一化、CSI 误差、UE 度数、总增益 |

训练目标包含两层约束：

| 目标 | 作用 |
|---|---|
| `rho_WMMSE` | 学习 WMMSE 的实际功率矩阵 |
| `y_share` | 学习每个 AP 内部的用户功率份额，强化按 AP 归一化后的相对分配 |

推理端 `computeRhoGNN.m` 会构造同样的节点特征，并调用 Python 推理模块生成 `rho`，最后按 AP 功率约束做归一化。

## 五、缓存与可重复性

主仿真缓存分两层：

| 缓存 | 内容 |
|---|---|
| 场景缓存 | AP/UE 位置、D 矩阵、信道估计等 |
| SNR 缓存 | 各算法的功率、预编码、ESR、时间和通信开销 |

缓存指纹包含系统规模、SNR、CSI 误差、算法表和 GNN 模型文件信息。更换模型或修改算法组合后，相关 SNR 缓存会自动失效。

## 六、绘图逻辑

当前 `plotESRResults_v2.m` 不再依赖固定行号，而是根据 `algoTable.pa`、`algoTable.pc` 和 `algoTable.mode` 精确筛选数据，避免算法表调整后画错曲线。

输出图包括：

| 图 | 关注点 |
|---|---|
| `Fig1_Best_PA_ESR` | 每种功率分配方法在所有 PC/Mode 下的最佳 ESR |
| `Fig2_RMMSE_PA_Comparison` | 固定 R-MMSE 后公平比较 PA |
| `Fig3_GNN_WMMSE_Gap` | GNN 相对 WMMSE 的差距 |
| `Fig4_Method_Summary` | 方法摘要与峰值 ESR |
| `Fig5_Time_Overhead_Comparison` | WMMSE 与 GNN 的端到端时间、核心计算时间和加速比 |

## 七、消融实验

当前消融图只保留 GNN 相关组件对比：

| 对比 | 目的 |
|---|---|
| GNN vs MLP | 验证图消息传递是否有效 |
| Per-sample vs Global | 验证归一化策略 |

## 八、设计重点

1. GNN 必须学习 WMMSE 的非均匀功率结构，而不是退化成 EPA。
2. 训练端和 MATLAB 推理端必须使用完全一致的特征 schema。
3. 绘图必须从算法表读取语义标签，不能依赖旧行号。
4. 缓存版本需要随着算法表和特征 schema 变化而更新。
