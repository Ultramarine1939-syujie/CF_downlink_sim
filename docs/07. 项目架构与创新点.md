# 项目架构与创新点

> 最后更新：2026-05-12 | 当前范围：GNN 功率分配，不包含协作聚合训练模块

## 一、系统总览

本项目是一个 Cell-Free Massive MIMO 下行链路功率分配与预编码仿真平台。当前核心目标是：用 GNN 学习 WMMSE 的功率分配规律，在 MATLAB 主仿真中以低延迟推理替代迭代式功率优化。

```mermaid
flowchart LR
    A["MATLAB 信道/拓扑生成"] --> B["WMMSE 标签生成"]
    B --> C[".mat 训练数据"]
    C --> D["Python GAT-GNN 训练"]
    D --> E["best_gat_gnn_power.pt"]
    E --> F["MATLAB 调 Python 推理"]
    F --> G["ESR 评估与绘图"]
```

## 二、目录分工

| 目录 | 职责 |
|---|---|
| `main/` | 主仿真入口、缓存、SNR 扫描和结果聚合 |
| `channel/` | Cell-Free 拓扑、信道估计和空间相关矩阵 |
| `precoding/` | MR、L-MMSE、R-MMSE、L-MMSE-G 预编码 |
| `power_allocation/` | Baseline、EPA、WMMSE、PSO、GNN 功率分配 |
| `python/` | GNN 数据集、训练、推理封装 |
| `visualization/` | ESR 曲线、PA 对比、消融图和结果打印 |
| `config/` | 默认参数与日志工具 |
| `docs/` | 项目说明文档 |

## 三、当前算法表

主仿真遍历 **64 种组合 = 8 PA x 4 PC x 2 Mode**。

功率分配：

| 方法 | 类型 | 作用 |
|---|---|---|
| Baseline | 传统 | 大尺度增益/距离基线 |
| Random | 传统 | 随机功率后归一化 |
| EPA | 传统 | 等功率分配 |
| D-WMMSE | 分布式优化 | 固定轮次消息交换的 WMMSE 近似 |
| Local-GNN | 分布式智能 | 每个 AP 仅基于本地 AP-UE 行独立推理 |
| WMMSE | 迭代优化 | GNN 标签源与性能参考 |
| PSO | 群智能优化 | 非凸搜索参考 |
| GNN | 深度学习参考 | 全图快速推理 |

预编码：

| 方法 | 作用 |
|---|---|
| MR | 低复杂度基线 |
| L-MMSE | 分布式线性 MMSE |
| R-MMSE | 考虑 CSI 误差的鲁棒 MMSE |
| L-MMSE-G | 全局协方差基准 |

主仿真为 PC 和 PA 分别记录架构标签，并用 `isDistributed` 判断组合是否进入分布式主排名。WMMSE、PSO、L-MMSE-G 和当前全图 GNN 保留为参考方法；MR/L-MMSE/R-MMSE 与 EPA/Baseline/Random/D-WMMSE/Local-GNN 的组合构成分布式候选集合。

接入模式：

| 模式 | 作用 |
|---|---|
| All | 全 AP 服务 |
| DCC | 动态协作簇 |

## 四、GNN 设计

当前全图 GNN 的训练和推理都使用同一套特征协议：

| 节点 | 特征 |
|---|---|
| AP | 对 UE 的 masked sqrt gain、SNR 归一化、CSI 误差、AP 度数、总增益 |
| UE | 来自 AP 的 masked sqrt gain、SNR 归一化、CSI 误差、UE 度数、总增益 |

训练目标包含两层约束：

| 目标 | 作用 |
|---|---|
| `rho_WMMSE` | 学习 WMMSE 的实际功率矩阵 |
| `y_share` | 学习每个 AP 内部的用户功率份额，强化按 AP 归一化后的相对分配 |

推理端 `computeRhoGNN.m` 会构造同样的节点特征，并调用 Python 推理模块生成 `rho`，最后按 AP 功率约束做归一化。

AP-local GNN 作为严格分布式最小版本，使用 `train_gnn_local.py` 训练、`gnn_runtime_local.py` 推理、`computeRhoLocalGNN.m` 接入 MATLAB。每个样本是一条 AP 行：

| 输入 | 说明 |
|---|---|
| masked gain row | `sqrt(gainOverNoise(l,:)) .* D(l,:)` |
| local D row | 本 AP 的 AP-UE 服务关系 |
| scalar context | SNR、CSI 误差、AP 度数、局部增益统计 |
| target | WMMSE 在本 AP 内的功率份额 `y_share(l,:)` |

## 五、缓存与可重复性

主仿真缓存分两层：

| 缓存 | 内容 |
|---|---|
| 场景缓存 | AP/UE 位置、D 矩阵、信道估计等 |
| SNR 缓存 | 各算法的功率、预编码、ESR、时间和通信开销 |

缓存指纹包含系统规模、SNR、CSI 误差、算法表和 GNN 模型文件信息。更换模型或修改算法组合后，相关 SNR 缓存会自动失效。

## 六、绘图逻辑

当前 `plotESRResults_v2.m` 不再依赖固定行号，而是根据 `algoTable.pa`、`algoTable.pc` 和 `algoTable.mode` 精确筛选数据，避免算法表调整后画错曲线。

输出图包括：

| 图 | 关注点 |
|---|---|
| `Fig1_Best_PA_ESR` | 每种功率分配方法在所有 PC/Mode 下的最佳 ESR |
| `Fig2_RMMSE_PA_Comparison` | 固定 R-MMSE 后公平比较 PA |
| `Fig3_GNN_WMMSE_Gap` | GNN 相对 WMMSE 的差距 |
| `Fig4_Method_Summary` | 方法摘要与峰值 ESR |
| `Fig5_Time_Overhead_Comparison` | WMMSE 与 GNN 的端到端时间、核心计算时间和加速比 |
| `FigA5_SyncLatency_ESR_Pareto` | PA+PC 组合的 ESR-同步时延 Pareto 消融 |
| `FigA6_PA_PC_SyncLatency_Heatmap` | PA×PC 同步时延矩阵 |
| `FigA7_RMMSE_PA_Latency_Ablation` | 固定 R-MMSE 下不同 PA 的时延/ESR 对照 |

## 七、消融实验

当前消融分为两类。

GNN 组件消融：

| 对比 | 目的 |
|---|---|
| GNN vs MLP | 验证图消息传递是否有效 |
| Per-sample vs Global | 验证归一化策略 |
| With vs No Augment | 验证数据增强 |

同步时延消融：

| 维度 | 目的 |
|---|---|
| PA 方法 | 比较 GNN/EPA/Baseline 的无迭代同步，与 WMMSE/PSO 的多轮同步代价 |
| PC 方法 | 比较 MR/L-MMSE 的低协同开销、R-MMSE 的鲁棒迭代开销和 L-MMSE-G 的集中式 CSI 开销 |
| AP 模式 | 通过 All/DCC 载荷比例体现动态协作簇对同步载荷的削减 |

同步消融结果导出到 `main/SimulationData/Sync_Ablation_Table.csv` 和
`main/SimulationData/Sync_Ablation_Results.mat`。模型参数在
`main/Combined_Downlink_Sim.m` 的 `syncAblation` 配置块中调整。

## 八、设计重点

1. GNN 必须学习 WMMSE 的非均匀功率结构，而不是退化成 EPA。
2. 训练端和 MATLAB 推理端必须使用完全一致的特征 schema。
3. 绘图必须从算法表读取语义标签，不能依赖旧行号。
4. 缓存版本需要随着算法表和特征 schema 变化而更新。
