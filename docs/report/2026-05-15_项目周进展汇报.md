# CF_downlink_sim 项目周进展汇报

汇报周期：2026-05-11 至 2026-05-15  
汇报主题：Cell-Free Massive MIMO 下行链路中分布式功率分配与 GNN 推理链路完善

## 1. 本周总体进展

本周项目主线从“全图 GNN 作为低时延参考方法”进一步推进到“具备严格分布式约束的 AP-local GNN 基线”。代码层面完成了分布式功率分配算法表重构、标准 WMMSE 标签生成、Local-GNN 训练与 MATLAB 推理接入、同步时延消融可视化，以及统一参数配置入口。实验层面已生成新训练数据、训练得到全图 GNN 与 Local-GNN 模型，并完成一轮 10 场景仿真与同步时延消融结果导出。

当前项目已经可以同时比较传统基线、D-WMMSE、集中式 WMMSE、全图 GNN 参考和严格 AP-local Local-GNN，并能够在结果表中区分“分布式候选方案”和“集中式/离线参考方案”。

## 2. 本周完成的主要工作

### 2.1 清理旧联邦学习实现，重新聚焦分布式下行功率分配

- 删除了早期联邦学习相关实现与消融脚本，避免把未成型的 FL 实现混入当前主线。
- 将全图 GNN 明确标记为 `low_latency_centralized` 参考方法，不再把它误归类为严格分布式方案。
- 重新梳理 Edge-FL-GNN 技术路线，形成新的任务规划文档，明确：Local-GNN 是当前已落地的严格分布式零消息基线，Neighbor-GNN、Edge-GNN 和 FL 训练仍属于后续阶段。

### 2.2 将 WMMSE 标签生成升级为标准 WMMSE

- 将原先简化版 WMMSE 功率分配替换为标准迭代 WMMSE 形式。
- 在 `computeRhoWMMSE.m` 中保留每 AP 功率约束，并输出迭代过程中的收敛信息。
- 该改动使 GNN 训练标签更加可靠，后续 Local-GNN、Neighbor-GNN 或 Edge-GNN 的监督目标更有说服力。

### 2.3 新增分布式功率分配基线 D-WMMSE

- 新增 `computeRhoDistributedWMMSE.m`，作为固定消息交换轮次的分布式优化基线。
- 在主仿真中将 D-WMMSE 标记为 `distributed`，可与 EPA、Baseline、Random、Local-GNN 一起进入分布式主排名。
- 同步时延模型中显式计入 D-WMMSE 的消息轮次、同步字节数和控制时延。

### 2.4 接入 AP-local Local-GNN 最小分布式版本

- 新增 `python/train_gnn_local.py`，用于训练 AP-local 功率份额预测模型。
- 新增 `python/gnn_runtime_local.py`，用于 MATLAB 调用 Python 进行 Local-GNN 推理。
- 新增 `power_allocation/computeRhoLocalGNN.m`，将 Local-GNN 接入主仿真功率分配流程。
- Local-GNN 每个 AP 只使用本地 AP-UE 行信息独立推理，不需要收集完整 `L x K` 全局图，因此满足当前严格分布式最小版本的约束。
- 当 `models/best_local_gnn_power.pt` 缺失或推理失败时，流程会自动回退到 EPA，保证主仿真不中断。

### 2.5 重构 GNN 推理计时与绘图工作流

- 新增 `python/gnn_runtime.py`，将全图 GNN 推理运行时从训练流程中解耦。
- 主仿真在正式计时前预热 GNN 和 Local-GNN 运行时，减少 Python 初始化对推理耗时统计的干扰。
- 更新 ESR 曲线、GNN-WMMSE 差距图、方法摘要图和计算时间对比图。
- 修复部分图例显示问题，并补充更具说明性的可视化输出。

### 2.6 新增同步时延消融与 A5 排名图

本周新增或完善了同步时延维度的评估：

- `FigA5_2_PA_Best_ESR_Ranking`：按每类功率分配方法的最佳 ESR 排名。
- `FigA6_PA_PC_SyncLatency_Heatmap`：预编码与功率分配组合的同步时延热力图。
- `FigA7_RMMSE_PA_Latency_Ablation`：固定 R-MMSE 后比较不同功率分配方法的 ESR-时延折中。
- `FigA8_LMMSE_PA_Latency_Ablation`：固定 L-MMSE 后比较不同功率分配方法的 ESR-时延折中。
- 导出 `Sync_Ablation_Table.csv`，包含每个算法组合的平均 ESR、同步轮次、同步字节数、同步时延和控制时延。

### 2.7 统一实验参数入口

- 新增并完善 `config/getDefaultParams.m`，将系统规模、SNR、CSI 误差、仿真规模、WMMSE 参数、D-WMMSE 参数、GNN 模型路径、训练数据参数和同步时延参数集中管理。
- `main/Combined_Downlink_Sim.m` 与 `power_allocation/exportTrainingData.m` 均改为读取统一配置。
- README 和运行指南同步改写，减少后续调参时多个文件不一致的问题。

### 3.2 主仿真配置

当前默认配置为：

- AP 数量：`L = 100`
- UE 数量：`K = 20`
- 每 AP 天线数：`N = 1`
- SNR 扫描：`5:5:30 dB`
- 场景数：`10`
- 每场景信道实现数：`200`
- 接入模式：`All` 与 `DCC`
- 算法组合数：`56 = 7 PA x 4 PC x 2 Mode`

### 3.3 同步时延消融关键结果

根据 `main/SimulationData/Sync_Ablation_Table.csv`，本轮共导出 56 个算法组合，其中 30 个为分布式候选，26 个为集中式或参考方案。

分布式候选中，平均 ESR 排名前列结果如下：

| 排名 | 算法组合 | 平均 ESR | 同步时延 ms | 控制时延 ms | 同步字节数 | 同步轮次 |
|---:|---|---:|---:|---:|---:|---:|
| 1 | Local-GNN + R-MMSE (DCC) | 20.342 | 0.4740 | 0.7169 | 28000 | 5 |
| 2 | Local-GNN + R-MMSE (All) | 19.859 | 0.8900 | 1.1189 | 80000 | 5 |
| 3 | Local-GNN + L-MMSE (DCC) | 19.547 | 0.0948 | 0.3377 | 5600 | 1 |
| 4 | D-WMMSE + R-MMSE (DCC) | 19.404 | 0.7304 | 0.7769 | 28800 | 10 |
| 5 | D-WMMSE + R-MMSE (All) | 18.986 | 1.1464 | 1.1789 | 80800 | 10 |
| 6 | Baseline + R-MMSE (DCC) | 18.900 | 0.4740 | 0.4939 | 28000 | 5 |

按每类功率分配方法的最佳结果看：

| 功率分配方法 | 最佳组合 | 平均 ESR | 是否分布式 | 同步时延 ms |
|---|---|---:|---:|---:|
| WMMSE | WMMSE + L-MMSE (DCC) | 22.184 | 否 | 2.8868 |
| Local-GNN | Local-GNN + R-MMSE (DCC) | 20.342 | 是 | 0.4740 |
| GNN | GNN + R-MMSE (DCC) | 20.179 | 否 | 0.4740 |
| D-WMMSE | D-WMMSE + R-MMSE (DCC) | 19.404 | 是 | 0.7304 |
| Baseline | Baseline + R-MMSE (DCC) | 18.900 | 是 | 0.4740 |
| Random | Random + R-MMSE (DCC) | 16.335 | 是 | 0.4740 |
| EPA | EPA + R-MMSE (DCC) | 7.381 | 是 | 0.4740 |

从当前结果看，Local-GNN 已经在分布式候选中取得最高平均 ESR，并且相对 D-WMMSE 在 ESR 上略高、同步轮次更少。集中式 WMMSE 仍然是性能上界，但控制时延明显高于 Local-GNN。因此，本周结果初步支持“用 AP-local GNN 作为低同步时延分布式功率分配基线”的技术路线。

## 5. 当前问题与风险

- Local-GNN 虽然满足严格 AP-local 约束，但只使用单 AP 的本地 AP-UE 行信息，尚未引入邻域消息，因此性能上限可能受限。
- 当前全图 GNN 仍依赖完整 `L x K` 图，只能作为低时延集中式参考，不能作为严格分布式主方案。
- Edge-GNN、Neighbor-GNN 与联邦训练尚未实现，Edge-FL-GNN 目前仍是规划路线，不应在汇报中表述为已完成。
- 训练数据与主仿真均已跑通，但后续还需要更多随机种子、更多场景规模和不同 CSI 误差强度下的鲁棒性验证。
- 代码工作区仍有部分未提交修改，建议汇报前整理一次提交记录，避免代码状态与汇报口径不一致。
