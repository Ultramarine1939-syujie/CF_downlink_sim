# 消融实验设计

> **项目名称**：CF_downlink_sim — Cell-Free 大规模 MIMO 下行链路仿真系统
> **最后更新**：2026-05-10
> **用途**：消融实验 (Ablation Study) 的完整设计文档，支撑论文实验章节

---

## 一、实验目的

通过系统地移除或替换各核心组件，验证每个组件对系统性能的独立贡献。消融实验结果用于论文中的 Ablation Study 章节，回答以下关键问题：

1. 图结构（GNN）是否真的有必要？纯 MLP 能否替代？
2. 注意力机制（GAT）比简单 MLP 带来了多少增益？
3. 联邦学习（FedAvg）相比集中式训练提升了多少？
4. 样本内归一化策略的优势有多大？
5. 数据增强对模型泛化能力的影响？
6. L-MMSE 分布式改造造成了多少信息损失？

---

## 二、消融维度总览

| # | 维度 | 对比组 | 验证假设 | 实验端 |
|---|------|--------|---------|--------|
| 1 | GNN vs No-GNN | PowerGNN_GAT vs PowerGNN_MLP | 图消息传递捕捉 AP-UE 拓扑关系的必要性 | Python |
| 2 | GAT vs MLP | PowerGNN_GAT vs PowerGNN_MLP | 多头注意力机制 vs 简单 MLP 的增益 | Python |
| 3 | FedAvg vs 集中式 | FedAvg 训练 vs 集中式训练 | 联邦学习弥补分布式信息损失的效果 | Python |
| 4 | 归一化策略 | Per-sample signed-log+minmax vs Global minmax | 样本内归一化对稀疏标签的有效性 | Python |
| 5 | 数据增强 | With vs Without 拓扑/噪声扰动 | 增强策略对模型泛化能力的贡献 | MATLAB+Python |
| 6 | L-MMSE 重构 | 全局协方差 vs 本地协方差 | 分布式预编码改造带来的 ESR 损失 | MATLAB |

## 三、实验配置

## 五、运行命令

### 前提条件

1. 已完成训练数据导出（`exportTrainingData.m`）
2. 已安装 Python 依赖（`pip install -r requirements.txt`）

### 运行全部消融实验

```bash
cd python
python train_gnn_ablation.py --ablation all --data "../data/gnn_training/*.mat"
```

### 运行单个消融维度

```bash
# 维度 1+2: GNN vs No-GNN
python train_gnn_ablation.py --ablation gnn_vs_nognn --data "../data/gnn_training/*.mat"

# 维度 3: FedAvg vs 集中式
python train_gnn_ablation.py --ablation fedavg_vs_central --data "../data/gnn_training/*.mat"

# 维度 4: 归一化策略
python train_gnn_ablation.py --ablation norm --data "../data/gnn_training/*.mat"

# 维度 5: 数据增强
python train_gnn_ablation.py --ablation augment --data "../data/gnn_training/*.mat"

# 维度 6: L-MMSE（打印 MATLAB 操作指南）
python train_gnn_ablation.py --ablation lmmse
```

### 快速测试（少量 epoch）

```bash
python train_gnn_ablation.py --ablation all --epochs 10 --fedavg_rounds 3
```

### 自定义参数

```bash
python train_gnn_ablation.py \
  --ablation all \
  --data "../data/gnn_training/*.mat" \
  --epochs 200 \
  --batch_size 64 \
  --hidden_dim 128 \
  --fedavg_rounds 30 \
  --output_dir "../models/ablation"
```

## 七、结果解读指南

### 7.1 核心指标

- **Val Corr**：预测值与真实值的相关系数（越高越好，接近 1.0 说明模型学到了 WMMSE 的分配规律）
- **Val NZ-MSE**：非零 rho 位置的 MSE（越低越好）
- **ESR Loss**：在 MATLAB 仿真中 GNN 方案相对于 WMMSE 的 ESR 损失比例

### 7.2 各维度预期结论

| 维度 | 预期结果 | 解读 |
|------|---------|------|
| GNN vs No-GNN | GAT Corr > MLP Corr | 图消息传递利用了 AP-UE 拓扑关系 |
| FedAvg vs 集中式 | FedAvg Corr >= 集中式 Corr | 联邦协作弥补了信息损失 |
| 归一化策略 | Per-sample Corr >> Global Corr | 样本内归一化解决了稀疏标签问题 |
| 数据增强 | With Augment Corr >= Without | 增强提升了拓扑泛化能力 |
| L-MMSE 重构 | ESR(L-MMSE-G) >= ESR(L-MMSE) | 分布式改造的 ESR 损失量化 |

## 九、注意事项

1. **数据增强消融的局限性**：当前增强策略在 MATLAB 端 `exportTrainingData.m` 中执行（离线增强），Python 端无法完全去除已嵌入的增强。严格消融需重新导出不含增强的数据集。

2. **随机性控制**：所有变体使用相同 seed（默认 42），确保数据划分和初始化一致。

3. **运行时间**：全部变体顺序执行约需 10-13 小时（CPU 环境），FedAvg 相关变体耗时最长。建议使用 `screen`/`tmux` 或后台运行。

4. **MATLAB 消融模型集成**：训练完消融模型后，将 `models/ablation/` 下的 `.pt` 文件放入 `models/ablation/` 对应子目录，`Combined_Downlink_Sim.m` 会自动检测并纳入仿真对比。

5. **L-MMSE 消融**：在 `Combined_Downlink_Sim.m` 的 PC 注册表中已添加 `LMMSE_G` 条目（`functionPrecoding_LMMSE_global`），会自动与分布式 `LMMSE` 对比。
