# 项目后续发展目标与论文化路线

> 最后更新：2026-05-15  
> 当前状态：已完成 WMMSE 标签生成、Full-Graph GNN、AP-local Local-GNN、D-WMMSE 分布式基线、同步时延消融和 10 场景初步实验。  
> 本文目标：基于当前代码与实验结果，规划后续要补强的实验、模型、理论分析和论文叙事。

## 1. 当前项目定位

当前项目已经从一个基础 Cell-Free Massive MIMO 下行仿真平台，推进到一个能够比较“集中式优化、分布式优化、全图 GNN、严格本地 Local-GNN”的实验系统。

当前可用方法包括：

| 类别 | 方法 | 当前定位 |
|---|---|---|
| 简单基线 | EPA / Random / Baseline | 低复杂度参考 |
| 集中式强参考 | WMMSE | GNN 教师标签来源；不声称全局最优 |
| 分布式优化基线 | D-WMMSE | 有限消息交换的分布式 WMMSE-style 近似 |
| 低时延集中式学习参考 | Full-Graph GNN | 使用完整 AP-UE 图，推理快但信息约束偏集中式 |
| 严格本地学习方法 | Local-GNN / AP-local MLP | 每个 AP 只使用本地 AP-UE 行信息，当前分布式主候选 |

当前最有潜力的论文叙事不是“提出一个复杂 GNN 架构”，而是：

> 将集中式 WMMSE 的功率分配经验蒸馏到严格本地的 AP-local 模型中，并在 Cell-Free Massive MIMO 下行链路中系统评估 ESR、同步时延、控制时延和通信开销之间的折中。

## 2. 总体发展目标

后续目标分为三个层次：

| 层次 | 目标 | 产出 |
|---|---|---|
| 实验可信度 | 证明当前 Local-GNN 优势不是偶然结果 | 规范数据划分、多随机种子、多场景泛化实验 |
| 方法创新性 | 从 Local-GNN 推进到 Neighbor-GNN / Edge-GNN | 更强的分布式 GNN 方法和 ESR-时延 Pareto 曲线 |
| 论文完整性 | 和已有 GNN 资源分配文献对比，补充复杂度与通信开销公式 | 可写成论文贡献点、实验表格和理论分析 |

## 3. 阶段 A：更规范的数据划分

### 3.1 问题

当前 Python 训练脚本主要采用随机划分：

```text
85% train
15% validation
```

这可以用于模型选择，但还不足以证明模型对未知网络拓扑、未知随机种子或未知 SNR 条件具有泛化能力。

### 3.2 后续目标

建立更清晰的数据划分方式：

| 划分类型 | 目的 | 示例 |
|---|---|---|
| Unseen seed | 测试不同随机信道实现下的稳定性 | train seeds: 1-8；test seeds: 9-10 |
| Unseen topology | 测试不同 AP/UE 空间分布下的泛化 | train topology set A；test topology set B |
| Unseen SNR | 测试 SNR 外推能力 | train: 5/10/15/20 dB；test: 25/30 dB |
| Unseen CSI error | 测试鲁棒性 | train: sigma_e=0.3；test: 0.1/0.5 |

### 3.3 建议实现

1. 在 `exportTrainingData.m` 中为每个 snapshot 保存更明确的 `seed`、`scenario_id`、`topology_id`、`SNR_dB`、`sigma_e`。
2. 在 Python 数据集加载器中增加按 meta 字段筛选的功能。
3. 增加固定划分配置文件，例如：

```text
config/splits/default_split.json
config/splits/unseen_snr_split.json
config/splits/unseen_topology_split.json
```

### 3.4 验收标准

- 能明确说出训练集、验证集、测试集分别来自哪些场景。
- 测试集不再只是随机切出来的一部分，而是具有明确泛化含义。
- 论文中可以报告 `seen` 与 `unseen` 两类结果。

### 3.5 当前实现状态

已新增可执行的数据划分框架：

```text
python/data_split.py
python/validate_splits.py
config/splits/default_split.json
config/splits/unseen_snr_split.json
config/splits/unseen_topology_split.json
config/splits/unseen_seed_split.json
config/splits/unseen_csi_error_split.json
```

训练脚本支持通过 `--split_config` 指定划分方式：

```bash
python python/train_gnn.py --data "data/gnn_training/gnn_training_data_20260514_214939.mat" --split_config config/splits/unseen_snr_split.json
python python/train_gnn_local.py --data "data/gnn_training/gnn_training_data_20260514_214939.mat" --split_config config/splits/unseen_seed_split.json
```

运行以下命令可在训练前验证所有划分配置：

```bash
python python/validate_splits.py
```

训练并评估某个 split 的系统级 ESR / 同步时延，可使用统一编排脚本：

```bash
# 仅训练 unseen_snr 的 Full-Graph GNN 与 Local-GNN
python python/split_system_eval.py --only unseen_snr --train both

# 使用已训练好的 split 模型启动 MATLAB 系统级仿真
python python/split_system_eval.py --only unseen_snr --eval --use-test-snr

# 快速冒烟评估：只跑 1 个场景、20 个信道实现
python python/split_system_eval.py --only unseen_seed --eval --num-scenarios 1 --num-realizations 20
```

该脚本默认使用最新的 `data/gnn_training/*.mat`，并将模型和结果分别写入：

```text
models/splits/<split_name>/
main/SplitEvaluations/<split_name>/Imgs/
main/SplitEvaluations/<split_name>/SimulationData/
```

MATLAB 主仿真已支持以下环境变量覆盖，因此不需要手动改 `getDefaultParams.m`：

| 环境变量 | 作用 |
|---|---|
| `CF_GNN_FULL_MODEL_FILE` | 指定 Full-Graph GNN checkpoint |
| `CF_GNN_LOCAL_MODEL_FILE` | 指定 Local-GNN checkpoint |
| `CF_OUTPUT_SAVE_PATH` | 指定图像输出目录 |
| `CF_OUTPUT_DATA_PATH` | 指定结果与缓存输出目录 |
| `CF_EVAL_TAG` | 写入结果中的评估标签 |
| `CF_SNR_DB` | 覆盖主仿真 SNR，例如 `"25 30"` |
| `CF_NUM_SCENARIOS` | 覆盖场景数 |
| `CF_NUM_REALIZATIONS` | 覆盖每场景信道实现数 |
| `CF_RUN_STAGE` | 覆盖仿真阶段，默认 3 |
| `CF_USE_CACHE` | 是否使用缓存 |

当前已有数据集上验证通过的划分包括：

| Split | Train | Val | Test | 说明 |
|---|---:|---:|---:|---|
| default | 5100 | 900 | 0 | 与原随机 train/val 行为兼容 |
| unseen_snr | 3400 | 600 | 2000 | train: 5-20 dB；test: 25-30 dB |
| unseen_topology | 4080 | 720 | 1200 | 每个 SNR/mode 内保留 snapIdx 401-500 作为测试 |
| unseen_seed | 4080 | 720 | 1200 | 每个 SNR/mode 内保留每 5 个快照中的 1 个作为测试 |
| unseen_csi_error | 3060 | 540 | 2400 | train: sigma_e 中间 60%；test: 低/高 20% 尾部 |

注意：当前 `unseen_csi_error` 主要利用现有 `sigma_e=0.3 ± 20%` 的增强范围。若要做更强的 CSI error 外推实验，仍建议重新生成包含多个基础 `sigma_e` 值的数据集，例如 `0.1 / 0.3 / 0.5`。

## 4. 阶段 B：多随机种子实验

### 4.1 问题

当前 Local-GNN 在某一轮实验中表现优于 Full-Graph GNN，但这可能来自：

- 训练随机种子差异；
- checkpoint 选择差异；
- 数据划分偶然性；
- 当前场景数量有限；
- Full-Graph GNN 尚未充分训练。

因此不能直接断言 Local-GNN 理论上优于 Full-Graph GNN。

### 4.2 后续目标

对关键方法进行多随机种子重复实验：

```text
Full-Graph GNN
Local-GNN
D-WMMSE
EPA
Baseline
WMMSE
```

建议至少使用：

```text
training seeds: 3-5 个
simulation seeds: 5-10 个
topology seeds: 5-10 个
```

### 4.3 需要输出的统计量

| 指标 | 输出形式 |
|---|---|
| 平均 ESR | mean ± std |
| 控制时延 | mean ± std |
| 同步字节数 | mean |
| Local-GNN 相对 D-WMMSE 增益 | percentage / absolute gap |
| Local-GNN 相对 WMMSE 差距 | absolute gap |

### 4.4 验收标准

- Local-GNN 的优势在多个随机种子下仍然稳定。
- 若 Local-GNN 只在部分场景优于 Full-Graph GNN，需要解释条件和原因。
- 图表中增加误差条或置信区间。

## 5. 阶段 C：和已有 GNN 资源分配论文明确对比

### 5.1 当前项目可对比的已有方向

| 已有方向 | 典型特点 | 本项目可强调的差异 |
|---|---|---|
| Decentralized GNN resource allocation | 强调 GNN 的去中心化和置换等变性 | 本项目落在 CF-mMIMO 下行功率分配，并显式评估同步时延 |
| WMMSE-unfolding GNN | 将 WMMSE 迭代展开为可学习网络 | 本项目不是展开 WMMSE，而是做 WMMSE teacher 到 AP-local student 的蒸馏 |
| Cell-Free precoding GNN | 学习预编码矩阵或线性预编码策略 | 本项目聚焦功率分配，并与多种预编码组合评估 |
| Sparse / multidimensional GNN | 稀疏化图结构以降低复杂度 | 本项目强调严格本地信息约束和通信开销建模 |
| DNN-based power control | 用普通神经网络替代传统功控 | 本项目显式区分 Full-Graph GNN、Local-GNN、D-WMMSE 和集中式 WMMSE |

### 5.2 本项目可形成的对比口径

建议论文中将项目特色总结为：

1. **Strict-local deployment**：Local-GNN 在线阶段每个 AP 只使用本地 AP-UE 行信息。
2. **Teacher-student formulation**：WMMSE 作为离线教师，Local-GNN 学习其 per-AP power share。
3. **ESR-latency-communication evaluation**：不仅比较 ESR，还比较同步时延、控制时延、通信字节数和同步轮次。
4. **Fair baselines**：同时包含 WMMSE、D-WMMSE、Full-Graph GNN、EPA、Random、Baseline。
5. **Future extensibility**：Local-GNN 可自然扩展为 Neighbor-GNN / Edge-GNN，形成 0/1/2 轮消息交换的 Pareto 曲线。

### 5.3 验收标准

- 文档中形成一张“已有工作 vs 本项目”的表。
- 明确指出本项目不把 Full-Graph GNN 误称为严格分布式。
- 明确指出 Local-GNN 当前结构较简单，创新重点在部署约束和系统评估，而不是复杂网络层设计。

## 6. 阶段 D：消融实验

### 6.1 目标

证明模型中每个设计组件都确实有贡献，而不是偶然堆叠。

### 6.2 建议消融项

| 消融项 | 对照设置 | 观察问题 |
|---|---|---|
| 去掉 D mask | 输入不包含 AP-UE 服务关系 | 模型是否依赖连接结构 |
| 去掉 gain stats | 不输入 degree / gain_sum / gain_mean / gain_max | 本地统计量是否有效 |
| 不同 hidden_dim | 32 / 64 / 96 / 128 / 256 | 模型容量对 ESR 和过拟合的影响 |
| 不同训练标签 | raw rho / normalized rho / per-AP share | 哪种标签更适合 Local-GNN |
| 不同 loss | MSE / Huber / share loss / hybrid loss | 训练目标对最终 ESR 的影响 |
| Full-Graph vs Local | 完整图信息 vs 本地行信息 | 信息量与部署约束的折中 |
| 无 GAT attention | MLP baseline | 图消息传递是否对 Full-Graph GNN 有实际收益 |

### 6.3 推荐优先级

第一优先级：

```text
Local-GNN: full features vs no D mask vs no gain stats
Local-GNN: per-AP share label vs raw rho label
Full-Graph GNN vs MLP baseline
```

第二优先级：

```text
hidden_dim sweep
num_layers sweep
loss function sweep
```

### 6.4 验收标准

- 每个消融实验至少输出 ESR、control delay 和 validation loss。
- 能解释 Local-GNN 的哪些输入特征最关键。
- 能解释当前 Local-GNN 为什么在某些设置下优于 Full-Graph GNN。

## 7. 阶段 E：解释 Local-GNN 当前优于 Full-Graph GNN 的原因

### 7.1 当前现象

当前实验中，Local-GNN 在部分指标上甚至优于 Full-Graph GNN。理论上 Full-Graph GNN 可见信息更多，表达上限应不低于 Local-GNN。因此需要谨慎解释。

### 7.2 可能原因

| 可能原因 | 解释 |
---|---|
| 任务更简单 | Local-GNN 只学习每个 AP 内部功率份额，输出空间更小 |
| 样本数量更大 | 一个完整快照可拆成 L 个 AP-row 样本 |
| 标签形式更合适 | per-AP power share 比完整 raw rho 更稳定 |
| Full-Graph GNN 未充分训练 | 图模型更复杂，可能需要更多数据或调参 |
| 归一化与 loss 影响 | 当前归一化策略可能更适配 Local-GNN |
| 实验偶然性 | 当前结果可能受随机种子和测试场景影响 |

### 7.3 后续验证实验

1. 统一 Full-Graph GNN 和 Local-GNN 的训练数据来源、训练轮数和随机种子。
2. 增加 Full-Graph GNN 的模型容量和训练时长。
3. 对 Full-Graph GNN 也尝试 per-AP share loss。
4. 在 unseen topology / unseen SNR 下比较二者。
5. 输出多随机种子的平均值和方差。

### 7.4 推荐表述

当前汇报和论文中建议使用：

> Local-GNN 在当前配置下表现出很强竞争力，可能受益于更简单的 per-AP power share 学习目标和更严格的本地归纳偏置。后续需要通过多随机种子和泛化实验验证其优势是否稳定。

避免使用：

> Local-GNN 理论上优于 Full-Graph GNN。

## 8. 阶段 F：加入 Neighbor-GNN 或 Edge-GNN

### 8.1 为什么需要新方法

Local-GNN 的优点是零消息和严格本地，但缺点是看不到邻居 AP 的干扰状态。为了形成更强的论文方法，需要在 Local-GNN 基础上进一步加入有限邻域信息。

### 8.2 Neighbor-GNN 方向

Neighbor-GNN 的核心目标：

```text
在 1-2 轮邻域消息交换内，让每个 AP 看到有限范围内的干扰信息。
```

可能输入：

```text
AP 本地 gain row
D row
邻居 AP 的压缩统计量
邻居对共享 UE 的服务强度
SNR / sigma_e
```

推荐比较：

| 方法 | 消息轮次 | 预期定位 |
|---|---:|---|
| Local-GNN | 0 | 零消息本地基线 |
| Neighbor-GNN-1 | 1 | 少量邻域信息 |
| Neighbor-GNN-2 | 2 | 更强邻域建模 |
| D-WMMSE | 固定多轮 | 分布式优化基线 |
| WMMSE | 多轮集中式 | 强参考 |

### 8.3 Edge-GNN 方向

Edge-GNN 的核心目标：

```text
由 Edge CPU 负责局部 cluster 内 AP-UE 子图推理。
```

适合处理：

- AP 被划分为多个 cluster；
- 每个 Edge CPU 只看到本 cluster 或固定邻域；
- 对重叠 UE 或边界 AP 做功率合并；
- 在同步模型中计入 AP 到 Edge CPU 的通信开销。

### 8.4 推荐优先级

建议先做 Neighbor-GNN，再做 Edge-GNN：

1. Neighbor-GNN 与当前 Local-GNN 代码关系更近，改造成本低。
2. Neighbor-GNN 能自然形成 0/1/2 轮消息的 ESR-时延 Pareto 曲线。
3. Edge-GNN 更适合作为后续系统架构扩展。

### 8.5 验收标准

- Neighbor-GNN 相对 Local-GNN 提升 ESR。
- 同步时延仍显著低于集中式 WMMSE。
- 可以画出：

```text
ESR vs sync delay
ESR vs sync bytes
message rounds vs ESR
```

## 9. 阶段 G：复杂度和通信开销公式

### 9.1 问题

当前项目已有仿真表格和同步时延估计，但论文中还需要更清晰的公式化分析。

### 9.2 建议补充的公式

#### WMMSE

需要描述：

```text
计算复杂度：与迭代次数、AP 数 L、UE 数 K、预编码/功率更新矩阵运算有关
通信开销：每轮需要同步全局或近似全局干扰/权重/功率变量
```

可先写成阶数量级：

```text
O(I_wmmse * f(L, K, N))
```

其中 `I_wmmse` 是迭代轮数。

#### D-WMMSE

当前代码近似：

```text
计算复杂度：O(R_dwmmse * L * K)
通信开销：O(R_dwmmse * K)
```

其中 `R_dwmmse` 是固定消息交换轮次。

#### Full-Graph GNN

对于 GAT 类模型：

```text
计算复杂度约与 GAT 层数、边数、hidden_dim、attention heads 有关
通信开销：需要收集完整 AP-UE 图信息
```

可写成：

```text
O(T_gnn * |E| * H * A)
```

其中 `T_gnn` 为 GNN 层数，`|E|` 为 AP-UE 边数，`H` 为隐藏维度，`A` 为注意力头数。

#### Local-GNN

当前 Local-GNN 是 AP-local MLP：

```text
每 AP 计算复杂度：O(F * H + hidden layers)
全网计算复杂度：O(L * C_local)
在线通信开销：O(0) 或仅本地 CSI 获取开销
```

核心表述：

> Local-GNN 不需要 AP 间或 AP-CPU 间为功率分配额外同步全局 CSI，因此在线功率分配通信开销最低。

#### Neighbor-GNN

未来可写：

```text
通信开销：O(R_neighbor * |E_neighbor| * message_dim)
计算复杂度：O(R_neighbor * |E_neighbor| * H)
```

### 9.3 验收标准

- 论文中有一张复杂度对比表。
- 复杂度分析和仿真中的 `sync_bytes`、`sync_rounds` 口径一致。
- 不只说“GNN 快”，而能解释快在哪里、通信少在哪里。

## 11. 最终论文贡献点草案

如果后续补齐实验，可将论文贡献点组织为：

1. 提出一种面向 Cell-Free Massive MIMO 下行功率分配的 WMMSE-to-local neural distillation 框架，将集中式 WMMSE 的功率分配经验迁移到 AP-local 在线推理模型。
2. 构建 Full-Graph GNN、Local-GNN、D-WMMSE、WMMSE、EPA 等方法的统一评估平台，明确区分集中式参考、分布式优化基线和严格本地学习方法。
3. 从 ESR、同步时延、控制时延、同步字节数和同步轮次多个维度评估学习式功率分配方法，而不是只比较频谱效率。
4. 进一步扩展 Neighbor-GNN / Edge-GNN，给出有限消息交换下的 ESR-时延 Pareto 折中。

## 12. 当前最优先的三件事

1. **先补数据划分和多随机种子**：这是让当前 Local-GNN 结果站稳的基础。
2. **再做关键消融实验**：解释 Local-GNN 为什么有效，以及 Full-Graph GNN 为什么当前没有明显优势。
3. **最后推进 Neighbor-GNN**：这是从工程项目走向论文方法创新的关键一步。
