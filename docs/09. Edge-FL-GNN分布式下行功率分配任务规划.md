# Edge-FL-GNN 分布式下行功率分配任务规划

> 目标：在现有 CF-mMIMO downlink 仿真平台上，将当前全图 GNN 功率分配扩展为“边缘 CPU 推理 + 联邦学习训练 + 局部/邻域图信息约束”的分布式方案。
> 最后更新：2026-05-15；当前代码已接入 AP-local GNN 最小分布式版本，Edge/Neighbor/FL 仍属于后续规划。

## 1. 网络信息核查结论

本规划参考了三类外部信息：

| 方向 | 核查结论 | 对本项目的含义 |
|---|---|---|
| 联邦学习 | FedAvg 的经典定义是让数据保留在本地节点，只上传本地模型更新并做模型平均，而不是上传原始数据 | 不能写“用联邦学习交换全体 CSI”；应写“交换模型参数/梯度，避免共享原始 CSI” |
| 边缘计算/边缘智能 | 边缘侧计算可降低数据回传和云端集中处理压力，适合低时延推理 | 引入 Edge CPU 做 cluster/neighbor 图推理是合理的 |
| GNN 无线资源分配 | GNN 已被用于学习无线资源分配，并利用图结构的置换等变性/局部聚合特性支持动态网络 | 将 AP-UE 关联建模成图，并在局部邻域上做推理是合理路线 |

关键参考：

- McMahan et al., 2017, [Communication-Efficient Learning of Deep Networks from Decentralized Data](https://proceedings.mlr.press/v54/mcmahan17a)：提出 FedAvg，核心思想是本地数据不上传，聚合本地更新。
- Wang, Eisen, Ribeiro, 2022, [Learning Decentralized Wireless Resource Allocations With Graph Neural Networks](https://arxiv.org/abs/2107.01489)：说明 GNN 可用于去中心化无线资源分配，并利用图结构适应动态网络配置。
- Springer AI Review survey, 2024, [Federated learning data security and privacy-preserving in edge-IoT](https://link.springer.com/article/10.1007/s10462-024-10774-7)：总结 FL 在边缘网络中通过上传模型更新而非原始数据来保护隐私和降低数据集中化压力。
- Springer WCN, 2023, [Distributed MMSE-based uplink receive combining, downlink transmit precoding and optimal power allocation in cell-free massive MIMO systems](https://link.springer.com/article/10.1186/s13638-023-02251-6)：说明 CF-mMIMO 中分布式 MMSE 预编码/功控是合理研究方向。

因此，本项目应采用如下严谨表述：

```text
Edge CPU 负责局部/簇级 GNN 推理；
Federated Learning 负责跨 Edge CPU 的模型训练协同；
联邦阶段交换模型参数或梯度，不交换全体 CSI；
在线下行阶段只使用本地/邻域信道统计信息进行低时延功率分配。
```

## 2. 当前项目状态

当前 GNN 流程：

```text
MATLAB 生成 AP-UE 拓扑与信道统计
        ↓
WMMSE 生成 rho_WMMSE 标签
        ↓
Python PowerGNN_GAT 学习全图 rho 映射
Python LocalPowerMLP 学习每个 AP 内部功率份额 y_share
        ↓
MATLAB computeRhoGNN 调用 Python 全图推理
MATLAB computeRhoLocalGNN 调用 AP-local 推理
        ↓
输出 rho(L,K)
```

当前优点：

- 可用一次 GNN 前向推理替代 WMMSE 多轮迭代。
- 已有完整训练数据导出、训练、推理、ESR 评估链路。
- 已有同步时延消融图和架构标签。
- 已有 `Local-GNN` 严格分布式最小版本：`train_gnn_local.py`、`gnn_runtime_local.py`、`computeRhoLocalGNN.m`，并已在主仿真算法表中标记为 `distributed`。

当前不足：

- 全图 `GNN` 仍使用完整 `L×K` 图，因此只能作为 `low_latency_centralized` 参考，不进入严格分布式主排名。
- `Local-GNN` 虽已满足每 AP 本地推理约束，但只看单行 AP-UE 信息，性能需要通过主仿真结果与 Full-Graph GNN、D-WMMSE、EPA、Baseline 对比验证。
- Edge-GNN、Neighbor-GNN 与联邦训练脚本尚未实现。

## 3. 目标架构

推荐目标方案命名：

```text
FL-Edge-GNN Power Allocation
```

架构分三层：

| 层级 | 组件 | 职责 |
|---|---|---|
| AP 层 | AP / RU | 采集本地大尺度增益、DCC 服务关系、局部 CSI 统计 |
| Edge CPU 层 | cluster controller / DU | 构造局部或邻域 AP-UE 子图，执行 GNN 推理，输出本 cluster/AP 的功率分配 |
| 联邦聚合层 | FL server / higher-layer controller | 周期性聚合多个 Edge CPU 的模型更新，不收集原始 CSI |

在线推理流程：

```text
APs -> Edge CPU: 本地或邻域信道统计
Edge CPU: 运行 Edge-GNN / Neighbor-GNN
Edge CPU -> APs: 下发本 AP 的 rho(l,:)
APs: 执行分布式 downlink 预编码与发射
```

离线/慢时标训练流程：

```text
每个 Edge CPU 保留本地历史数据
        ↓
本地训练 Edge-GNN
        ↓
上传模型更新 Δθ 或 θ
        ↓
FL server 执行 FedAvg/FedProx
        ↓
下发全局模型
        ↓
Edge CPU 本地微调并部署
```

## 4. 技术路线

### 阶段 A：架构标签与基线清理

状态：已完成当前版本。

任务：

- 在主仿真中区分 `distributed candidate` 与 `centralized/offline reference`。
- 保留 WMMSE、L-MMSE-G 作为参考，不进入分布式主排名。
- 保留当前全图 GNN 作为 low-latency centralized reference。
- 新增 D-WMMSE 作为固定消息轮次的分布式优化基线。
- 新增 Local-GNN 作为严格 AP-local 分布式智能基线。

验收标准：

- `algoTable.isDistributed` 可用于筛选主排名。
- 主结果表不把 WMMSE/L-MMSE-G 当作分布式候选。
- `Sync_Ablation_Table.csv` 输出 `pc_arch`、`pa_arch`、`is_distributed`。

### 阶段 B：Edge-GNN 局部/簇级推理

任务：

1. 新增 MATLAB 接口：

```text
power_allocation/computeRhoEdgeGNN.m
```

2. 新增 Python 推理模块：

```text
python/gnn_runtime_edge.py
```

3. 将全局 `L×K` 输入切分为 cluster/neighbor 子图：

```text
cluster c:
  AP set: A_c
  UE set: U_c
  D_c = D(A_c, U_c)
  gain_c = gainOverNoise(A_c, U_c)
```

4. Edge CPU 输出局部功率：

```text
rho(A_c, U_c)
```

5. 对重叠 UE 或重叠 AP 的输出做一致性合并：

```text
同一 AP 的多个输出 -> 按 AP 行归一化
同一 UE 的多个 cluster 预测 -> 以 AP 所属 cluster 为准，或做加权平均
```

验收标准：

- 在线推理不再传入完整 `L×K` 图给单个模型。
- 每个 Edge CPU 只看到本 cluster 或固定邻域信息。
- 每 AP 满足功率约束。
- 与当前 GNN、D-WMMSE、EPA、Baseline 对比 ESR 和同步时延。

### 阶段 C：AP-local GNN 分布式最小版本

状态：已接入当前代码路径。

目的：先做一个严格分布式、实现成本低的版本。

已新增：

```text
python/train_gnn_local.py
python/gnn_runtime_local.py
power_allocation/computeRhoLocalGNN.m
```

样本定义：

```text
input_l:
  gainOverNoise(l,:) .* D(l,:)
  D(l,:)
  SNR
  sigma_e
  AP degree
  local gain statistics

target_l:
  y_share(l,:)  # WMMSE 在本 AP 内的归一化功率份额
```

推理：

```matlab
for l = 1:L
    rho(l,:) = localGNN(gainOverNoise(l,:), D(l,:), Pt, sigma_e);
end
```

验收标准：

- 每个 AP 独立推理。
- 不需要 Edge CPU 收集完整全局图。
- `paArch = distributed`，并进入分布式主排名。
- 缺少 `models/best_local_gnn_power.pt` 或推理失败时自动回退 EPA，保证仿真流程不中断。
- 后续需要用稳定训练结果验证 ESR 是否优于 EPA，并量化相对 Baseline/D-WMMSE/Full-Graph GNN 的差距。

### 阶段 D：Neighbor-GNN 固定轮次邻域消息

目的：在 Local-GNN 基础上提高性能。

邻域定义可选：

| 邻域方式 | 说明 |
|---|---|
| DCC 邻域 | 与同一 UE 关联的 AP 构成邻域 |
| 地理邻域 | 距离最近的若干 AP |
| 增益阈值邻域 | 大尺度增益超过阈值的 AP-UE 边 |
| Top-M 服务邻域 | 每个 UE 只保留最强 M 个 AP |

消息内容建议限制为标量或低维 embedding：

```text
per-UE aggregate gain
per-UE interference proxy
neighbor AP load
served UE count
```

验收标准：

- 固定消息轮次，例如 1 或 2 轮。
- 同步时延模型中显式计入 neighbor message bytes。
- ESR 高于 Local-GNN。
- 同步时延低于集中式 WMMSE。

### 阶段 E：联邦训练

新增：

```text
python/federated_train_edge_gnn.py
python/fl_clients/
python/fl_server/
```

数据划分：

```text
client = one Edge CPU / one geographic area / one scenario group
local data = snapshots generated by that client
```

推荐算法：

| 算法 | 用途 |
|---|---|
| FedAvg | 第一版，简单可靠 |
| FedProx | 处理非 IID client 数据 |
| FedAvg + local fine-tuning | 提升特定场景性能 |

训练流程：

```text
for round = 1:R:
    server 下发 θ_t
    selected clients 本地训练 E epochs
    clients 上传 Δθ 或 θ_i
    server 聚合 θ_{t+1}
```

禁止事项：

- 不上传原始 `Hhat`
- 不上传完整 `gainOverNoise`
- 不上传完整 AP-UE 样本库

验收标准：

- 能输出 `best_federated_edge_gnn.pt`。
- 与集中训练 Edge-GNN 对比：
  - ESR gap
  - 训练通信量
  - 不同 client 非 IID 条件下泛化性能
- 文档中明确 FL 交换模型更新，不交换 CSI。

## 5. 需要新增的实验

## 7. 风险与处理

| 风险 | 影响 | 处理 |
|---|---|---|
| Local-GNN 性能低 | 无法支撑主方案 | 使用 Neighbor-GNN 增加有限邻域信息 |
| FL 非 IID 收敛慢 | 联邦模型泛化差 | 引入 FedProx、client sampling、local fine-tuning |
| Edge CPU cluster 划分不稳定 | 结果波动 | 固定 DCC/距离/Top-M 三种划分并做消融 |
| 模型更新通信量仍大 | FL 训练开销高 | 降低聚合频率、模型压缩、只在慢时标训练 |
| 论文表述混淆 | 容易被质疑 | 明确区分 online inference 与 offline/federated training |

## 9. 论文可用表述

建议写法：

```text
We propose an edge-assisted federated GNN framework for distributed downlink
power allocation in CF-mMIMO systems. Edge CPUs perform low-latency GNN
inference based on local or neighborhood CSI statistics, while federated
learning is used only for slow-timescale model training across edge domains.
The raw CSI samples are kept locally and only model updates are exchanged,
thereby reducing online synchronization overhead and avoiding centralized CSI
collection.
```

中文表述：

```text
本文进一步设计边缘辅助的联邦 GNN 分布式功率分配框架。在线阶段，各边缘 CPU 仅基于本簇或固定邻域内的信道统计信息执行低时延 GNN 推理；离线或慢时标阶段，不同边缘 CPU 通过联邦学习交换模型参数更新，而不共享原始 CSI 数据。该框架兼顾了低同步时延、分布式部署和跨场景泛化能力。
```
