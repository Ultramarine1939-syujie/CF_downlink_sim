# 图像输出与 U-GNN 说明

本文说明 `main/Imgs/` 中当前生成的图像含义，并补充 U-GNN 模型的搭建方式及其与其他方法的区别。每张图通常同时保存 `.png` 和 `.fig`：`.png` 用于汇报展示，`.fig` 是 MATLAB 可编辑版本，二者内容一致。

## 图像说明

### Fig1_Best_PA_ESR

来源：`plotESRResults_v2.m`。

这张图展示每种功率分配方法在不同 SNR 下的最佳 ESR 曲线。对每个 PA 方法，程序会在候选预编码方法中选出 ESR 最好的组合，因此它回答的是“某个功率分配方法在最佳配套预编码下能达到什么性能”。横轴是 SNR，纵轴是平均 ESR。

### Fig2_RMMSE_PA_Comparison

来源：`plotESRResults_v2.m`。

这张图固定预编码为 R-MMSE，只比较不同功率分配方法的 ESR 随 SNR 变化。它更适合观察 PA 方法本身的差异，因为预编码方法不再变化。

### Fig3_GNN_WMMSE_Gap

来源：`plotESRResults_v2.m`。

这张图比较学习型方法与 WMMSE 的性能差距。通常以 R-MMSE+DCC 场景为基准，计算 GNN、DCGNN、U-GNN、DQN、DDPG 等方法相对 WMMSE 的 ESR 百分比差距。它主要用于说明学习方法距离高性能迭代优化方法还有多少差距。

### Fig4_Method_Summary

来源：`plotESRResults_v2.m`。

这是一张方法汇总图，用紧凑表格或摘要形式展示主要方法的平均性能、计算时间或速度优势。它适合放在汇报中作为总览页，帮助快速比较“性能”和“开销”的综合表现。

### Fig5_Time_Overhead_Comparison

来源：`plotESRResults_v2.m`。

这张图展示 WMMSE 与学习型方法的时间开销对比，重点反映学习型推理相比迭代 WMMSE 的低时延优势。图中关注的是端到端或核心计算时间，以及由此得到的速度提升。

### FigA5_2_PA_Best_ESR_Ranking

来源：`plotLatencyAblationResults.m`。

这张图在 DCC 和分布式预编码约束下，为每种 PA 方法选出 ESR 最好的 PA+PC 组合，并按 ESR 从高到低排序。柱高表示平均 ESR，柱上文字给出该组合的 control delay 和 compute delay，因此它同时展示性能排序和控制时延代价。

### FigA6_PA_PC_SyncLatency_Heatmap

来源：`plotLatencyAblationResults.m`。

这张热力图展示 DCC 场景下不同 PA 与 PC 组合的控制时延。横轴是预编码方法，纵轴是功率分配方法，格子中的数字是 control delay，颜色是 `log10(control delay + 1e-3)`。它适合观察哪些 PA+PC 组合会带来较大的控制链路开销。

### FigA7_RMMSE_PA_Latency_Ablation

来源：`plotLatencyAblationResults.m`。

这张图固定预编码为 R-MMSE，对比不同 PA 方法的控制时延和平均 ESR。左轴柱状图是 control delay，右轴曲线是平均 ESR。它回答的是：在 R-MMSE 下，不同功率分配方法的性能和时延如何权衡。

### FigA8_LMMSE_PA_Latency_Ablation

来源：`plotLatencyAblationResults.m`。

这张图固定预编码为 L-MMSE，展示不同 PA 方法的 control delay 和平均 ESR。读法与 FigA7 相同，只是预编码从 R-MMSE 换成 L-MMSE。

### FigA9_MR_PA_Latency_Ablation

来源：`plotLatencyAblationResults.m`。

这张图固定预编码为 MR，展示不同 PA 方法的 control delay 和平均 ESR。它用于观察在更简单的 MR 预编码下，功率分配方法的性能收益是否仍值得其控制时延开销。

## FigA7 到 FigA9 的时延口径

FigA7、FigA8、FigA9 使用的是 `control_delay_ms`，不是单独的 `sync_delay_ms`。当前口径可以概括为：

```text
control_delay_ms = sync_delay_ms + PA compute delay + PC compute delay
sync_delay_ms = sync rounds * RTT + sync/feature bytes / fronthaul rate
```

其中 WMMSE/D-WMMSE 计入迭代同步开销；Local-GNN、EPA、FPCP 近似认为没有额外 PA 同步开销；GNN、U-GNN、DQN、DDPG 没有多轮 PA 迭代同步，但会计入一次特征收集或集中推理通信开销。

## U-GNN 模型如何搭建

U-GNN 的核心网络结构复用了 `PowerGNN_GAT`，也就是和监督式 GNN 相同的 AP-UE 图神经网络骨干。每个样本被构造成 AP 节点和 UE 节点组成的二部图：AP 节点特征包含 masked sqrt(gain)、SNR、CSI、连接度和增益上下文；UE 节点也包含对应的信道和上下文特征。模型先对 AP/UE 节点分别做 LayerNorm 和线性嵌入，然后经过多层 GATConv 消息传递，并使用残差连接和 LayerNorm 稳定训练。最后只取 AP 节点隐藏状态，拼接 AP 原始特征后通过 AP 级预测头输出每个 AP 给各 UE 的功率份额 logits。

U-GNN 的关键区别在训练目标而不是图网络骨干。普通 GNN/DCGNN 依赖 WMMSE 生成的标签，训练时学习拟合 WMMSE 的功率分配；U-GNN 不使用 WMMSE 标签，而是把模型输出转换为每个 AP 的功率份额，再通过可微的大尺度等效 sum-rate 代理函数直接最大化 ESR。因此 U-GNN 是 teacher-free 的无监督学习方法。

当前 U-GNN 默认使用 `share_logits_mix` 输出形式。模型除了输出 K 个 UE 的 neural share logits 外，还为每个 AP 输出一个混合门控系数 `lambda`，把神经网络策略和 FPCP 解析先验做软混合：

```text
share = lambda * share_gnn + (1 - lambda) * share_fpcp
```

这个 FPCP 先验是训练和推理中的软约束，不是简单的硬回退。训练损失中还包含 baseline guard、anchor、entropy floor、consistency regularization 等项，目的是防止无监督代理函数把功率分配推向过度集中的一热点解。

## 与其他方法的区别

- WMMSE/D-WMMSE：迭代优化方法，通常性能强，但需要多轮迭代和同步，控制时延较高。
- GNN：监督式集中 GNN，使用 WMMSE 标签训练，推理速度快，但目标是模仿 WMMSE。
- DCGNN：与 GNN 类似，也是监督式学习，但图结构更偏动态或局部连接，强调分布式/动态拓扑适配。
- Local-GNN：更偏本地低时延推理，通信和同步开销更低，但可用全局信息较少。
- U-GNN：网络骨干接近 GNN，但不依赖 WMMSE 标签，直接用可微 ESR 代理训练；同时通过 FPCP 软先验和安全正则降低无监督训练的不稳定性。
- DQN/DDPG：强化学习式功率分配，训练范式不同；主仿真中也按集中式低时延学习方法处理，计入一次特征收集/集中推理通信开销。
- EPA/FPCP：解析式低复杂度方法，几乎没有 PA 同步开销，但性能上限通常低于 WMMSE 或训练充分的学习方法。

简言之，U-GNN 可以理解为“使用 GNN 结构、但不学 WMMSE 标签，而是直接朝 sum-rate 目标优化”的方法。它试图在 WMMSE 的高性能和学习方法的低时延之间取得折中。
