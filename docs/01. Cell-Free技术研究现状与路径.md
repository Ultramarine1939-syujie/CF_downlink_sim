# 关于Cell-Free技术的研究现状与路径

Cell‑Free（无蜂窝网络）无蜂窝是一种新型的组网方式，是打破传统蜂窝边界、由大量分布式接入点（AP）协同服务用户的新型移动通信架构；对6G超高峰值速率、超高频谱效率、海量连接以及超低时延和超高可靠传输均有重要的支撑作用。其核心目标是通过大规模节点在同一时频资源上服务大量用户，将多小区干扰转化为有用信号，从而实现频谱效率和系统性能的持续提升。

无小区划分：**没有固定蜂窝边界**，用户移动时无需小区切换。

多 AP 协同：**大量小型 AP 密集部署**，由中心 / 边缘单元统一调度，**多 AP 同时为一个用户收发信号**。

联合处理：信号在中心 / 边缘节点**联合解码、联合预编码**，化干扰为增益。

## 研究背景

多天线技术及密集组网作为提高频谱效率的主要方法，在 3G~5G 中得到广泛应用，如今基站的发送天线数从 3G 应用的 2 根增加到 5G 应用的 64 甚至 128 根，小区分裂也从宏蜂窝、微蜂窝到微微蜂窝。然而，**集中式系统**中增大天线规模遇到的**物理实现问题**和小区分裂遇到的**干扰问题**，使得 5G 系统的**频谱效率不能持续性提升**。因此，需要打破传统蜂窝架构以及小区分裂的思维方式，采用新型的无蜂窝组网及相应的大规模协作 MIMO 传输技术。

另一方面，由于低频段频谱资源几近枯竭，拓展频谱资源是提高峰值速率最直接的方法，因而从 5G 的毫米波到更高频段的太赫兹成为满足 6G 峰值速率的主要解决手段。然而，**高频段**的**大带宽、近光学和易被遮挡**等特点，使其在移动通信应用中面临众多技术挑战。在无蜂窝架构下，通过协作传输，可以有效解决高频段易被遮挡的问题，提高链路的鲁棒性。

## 技术特点

无缝覆盖：**无切换、无边缘效应**，移动中速率更稳定

容量更高：**空间分集 + 全频复用**，频谱效率与用户公平性显著提升

低时延高可靠：**分布式 + 集中式混合处理**，适合工业、车联网等场景

部署灵活：**低成本小 AP 密集组网**，适配室内、密集城区、低空等复杂环境

| 对比项   | 传统蜂窝网络   | Cell‑Free 无蜂窝网络       |
| -------- | -------------- | -------------------------- |
| 服务主体 | 单基站服务     | **多 AP 协同服务**         |
| 覆盖边界 | 有明确小区边界 | **无边界、连续覆盖**       |
| 切换机制 | 小区间硬切换   | **无切换、无缝漫游**       |
| 干扰处理 | 小区间干扰严重 | **联合处理、化干扰为增益** |

## 研究路径

### 1. 核心基础（收发机架构设计、信道信息传递、频谱双工融合优化、资源分配调度体系）

#### 1.1 分布式收发机架构设计

从技术演进来看，分布式收发机架构以 5G mTRP 技术为核心基础，3GPP 在 R15 至 R19 阶段持续完善 mTRP 上下行传输方案，上行定义了单频网（SFN）和空分复用（SDM）传输模式，还新增仅上行接收的 TRP 并通过定时提前（TA）测量实现分集接收 [25]，将该方案推广至多用户场景并引入干扰消除技术，即可实现无蜂窝大规模 MIMO 的上行分布式多用户 MIMO 接收；下行则围绕非相干联合发送（NCJT）与相干联合发送（CJT）展开研究，R19 通过码本反馈机制结合时延、频偏及相位反馈，初步构建了 CJT 传输方案，但受终端能力限制仅支持最多 4 个 TRP 协作和 2 个 TCI 状态 [1]。同时，传统非码本 mTRP 多用户协作传输因终端信道估计能力限制，多子带共用预编码矩阵存在显著性能损失，而随着 TRP 配置数量增加，基带信号处理算力的需求也大幅提升 [1]。

未来的研究方向将聚焦于突破现有 mTRP 技术的碎片化限制，立足于 mTRP 架构综合考虑高低频段、不同 TRP 形态及终端能力多样性，构建更简洁高效的传输机制；同时结合 DMRS 设计适用于多用户场景的高性能协作预编码方案，解决多子带共用预编码矩阵的性能损耗问题 [1]；此外，学术界提出的可扩展分布式基带信号处理架构将成为研究重点，为基站产品性能提升提供理论与技术支撑，满足大规模 TRP 动态协作簇传输对算力的需求 [1]。

#### 1.2 上下行信道信息精准获取

在 TDD 系统的信道信息获取研究中，3GPP R19 版本首次引入终端辅助的互易性校准机制，提出两种获取 TRP 间校准系数的方法，一是基于上行 SRS 信道估计结合预编码 CSI-RS 实现终端侧测量反馈，二是由终端直接反馈下行信道统计特性供网络侧计算 [1]；针对低频段 TDD 系统，当 TRP 间存在空口链路且具备自校准能力时，可实现高性能多用户协作传输，同时 IEEE 1588 精准时间协议成为 TRP 间时频同步的核心技术，但协议在交换机级联场景下的同步精度会显著下降 [12]。此外，预编码 CSI-RS 校准机制还为终端多天线校准提供了可行路径，成为实现上行 CJT、提升上行覆盖性能的关键方法 [1]，而多用户 CJT 传输对信道相位变化敏感，需结合校准相位时刻、历史 CSI 及 QCL 关系通过预测获取高精度下行 CSI [12]。

未来需重点提升 IEEE 1588 协议在级联场景下的时间同步精度，满足大规模 TRP 协作对时频同步的严苛要求 [12]；同时深入研究网络侧自校准技术的工程实现方案，简化 QCL 与 TCI 的复杂流程，降低终端的测量与反馈开销 [12]；此外，针对多用户 CJT 传输的高精度 CSI 需求，需优化 CSI 预测算法，结合 AI 技术提升信道信息的实时性与准确性，减少信道信息过时对传输性能的影响 [1]；还可进一步拓展终端多天线校准机制的应用场景，实现多终端协同校准，为大规模上行 CJT 传输奠定基础。

#### 1.3 频谱与双工技术融合优化

传统 5G 载波聚合采用多小区管理方式，存在信令开销高、传输延迟大、频谱利用率低的问题，难以适配超密集用户场景，而 6G Day 1 技术讨论中提出的单小区 / 超小区多载波（SCMC）频谱融合技术，可通过灵活配置 TDD 上下行时隙配比，同时支撑大上行与大下行终端需求，与无蜂窝技术分别从频域和空域拓展传统小区概念 [6,13]。在双工技术方面，5G-A 引入的子带全双工（SBFD）技术可降低 TDD 系统传输时延并提升上行覆盖性能，而网络辅助全双工（NAFD）技术借助 mTRP 大规模天线阵列的空域隔离特性实现全双工传输，同时利用 TRP 协作能力抑制交叉链路干扰（CLI）[15]，频谱融合与 SBFD 技术均与 mTRP 设计理念相通，共同构成以用户为中心的空时频资源配置框架 [4]。

未来需对 SCMC 与无蜂窝技术进行协同设计，统一考虑初始接入、同步信号设计、传输方法及信令机制，推动两项技术的落地应用 [6]；同时深入研究中频段（FR3）SBFD 的部署方案，充分挖掘多天线系统的空域隔离潜力，提升全双工传输的稳定性 [14]；此外，需研究多节点 / 多通道双工模式动态选择方案，实现空口资源的灵活配置 [16]；还需解决频谱融合与 mTRP 技术带来的资源分配复杂度提升问题，在提升单用户体验的同时，保障系统总容量的持续增长 [4]。

#### 1.4 分布式调度与资源分配体系构建

传统蜂窝架构采用分布式调度方式，系统复杂度较低但多小区间缺乏联合调度与资源分配，容量提升效果有限；而集中式调度的无蜂窝基站虽能实现用户规模与系统容量同步增长，但实现复杂度高、可扩展性差 [10]。当前研究提出了基于 vCPU + 分布式 MAC（D-MAC）的无蜂窝调度与资源分配架构，在 vCPU 实现高阶物理层功能，在 D-MAC 层执行可扩展分布式资源分配，用户初始接入形成专属 TRP 集合并关联 D-MAC 实例，调度器通过实时信息交互抑制资源冲突干扰 [10]。同时，5G 演进标准中的 L1/L2 触发的移动性（LTM）机制可借助 MAC-CE 指令实现 TRP 集合动态选择，避免传统层 3 切换流程，降低移动性管理时延 [10]。

未来的核心研究方向是解决非理想回传条件下的 D-MAC 机制设计问题，优化 TRP 与 vCPU 间的信息交互策略，提升架构在实际组网中的适配性 [10]；同时结合 AI 技术完善终端移动性管理，通过用户移动轨迹预测优化 TRP 集合动态调整策略，进一步提升移动性管理效率 [10]；此外，需优化 D-MAC 调度器的资源协调机制，降低多实例间的信息交互开销，在保障调度精度的同时控制系统复杂度 [10]；还可研究分布式与集中式结合的混合调度方案，兼顾系统可扩展性与资源分配的最优性。

### 2. 技术痛点（干扰抑制优化、资源管理机制、多用户场景优化）

#### 2.1 广域组网异步干扰抑制与优化

当前移动通信系统沿用 OFDM 技术，受循环前缀（CP）时长制约，当终端与 TRP 间传播时延差超过 CP 范围时，会引发异步干扰并进一步导致载波间干扰（ICI），严重恶化组网性能，5G 典型子载波间隔对应的 CP 等效距离存在明显上限，单站覆盖能力越强，多站点协作的同步区域占比越小 [14]。在现有基站系统中，终端仅与单个 TRP 实现时间同步，上行方向远端 TRP 未调整接收窗、下行方向终端未实施干扰抵消，均会产生异步干扰 [18]，而密集城区场景下，大规模 MIMO 窄波束传输的反射径会引发远端异步干扰，成为网络性能提升的核心瓶颈 [18]。同时，传统大规模 MIMO 通过增加天线数量提升性能，但受导频资源开销及天线规模限制，多站点间干扰难以完美抑制，远端干扰问题突出 [18]。

未来需研究单用户场景下的 mTRP 异步协作优化方案，通过调整 TRP 接收窗、优化终端干扰抵消接收算法，在低复杂度下实现分集增益 [18]；针对多用户空分复用场景，需探索低代价的多用户异步干扰抵消技术，结合干扰抑制接收机设计，平衡性能与实现成本 [18]；同时重新优化 TRP 通道配置策略，通过部署小规模天线的模块化 TRP 结合空口校准机制，替代传统大规模 MIMO 的高通道配置，降低远端干扰并提升系统能效 [18]；此外，需研究适配广域无蜂窝组网的同步机制，通过局部同步结合干扰协调，扩大同步区域占比，减少异步干扰的产生。

#### 2.2 非理想回传条件下的资源管理机制

当前无蜂窝系统的分布式 MAC（D-MAC）架构仅在理想前传 / 回传条件及高效信息交互前提下，才能实现可扩展的资源分配，并与物理层分布式基带处理配合支撑无蜂窝组网 [10]。但在实际工程部署中，前传 / 回传链路存在时延、丢包、带宽受限等非理想问题，现有 D-MAC 机制缺乏相应的适配设计，成为无蜂窝技术从理论研究走向工程落地的重要障碍 [10]。同时，随着无蜂窝组网规模扩大，TRP 与 vCPU、D-MAC 实例间的信息交互量大幅增加，非理想回传会进一步加剧资源分配的延迟与误差，导致干扰抑制效果下降、系统性能受损 [10]。

未来需重点研发适配非理想回传条件的 D-MAC 调度算法，引入链路状态感知机制，根据回传链路的时延、丢包率动态调整信息交互策略与资源分配周期 [10]；同时结合边缘计算技术，将部分资源调度功能下沉至边缘分布式单元，减少核心网交互开销，降低回传链路压力 [10]；此外，需研究轻量化的资源分配信息交互协议，在保证调度精度的前提下，减少交互数据量，提升非理想回传下的信息传输效率 [10]；还可引入预测性资源分配机制，基于历史链路状态与用户需求，提前规划资源分配方案，降低回传非理想性对实时调度的影响。

#### 2.3 大规模用户场景的可扩展性优化

无蜂窝大规模 MIMO 的核心要求是在用户规模不断扩大时，网络侧收发节点及空时频资源的调度复杂度仅呈线性增长 [9]，而当前研究提出的可扩展分布式协作基带处理架构及调度与资源分配策略，为该目标的实现提供了理论基础 [9,10]。但在实际部署中，受成本、信道传播时延及异步问题限制，单个 TRP 同时服务的用户数、为一个用户服务的 TRP 数量均存在上限 [9]，且大规模用户下，TRP - 用户关联、用户配对、资源分配的耦合度大幅提升，导致调度复杂度难以控制在线性增长范围内 [10]。同时，传统资源分配策略在大规模用户场景下，难以实现用户需求与 TRP 资源的精准匹配，资源利用率与用户公平性难以兼顾 [9]。

未来需深入研究可扩展的 TRP - 用户动态关联机制，合理限制单个 TRP 服务用户数与单个调度器管理用户数，在性能与复杂度间寻求平衡 [9]；同时引入轻量化算法与 AI 辅助调度，通过强化学习实现大规模用户下的快速用户配对与资源分配，提升调度效率 [9]；此外，需研发分布式并行调度技术，将大规模用户调度任务拆分至多个 D-MAC 实例，实现并行处理，控制整体调度复杂度 [10]；还可研究基于用户业务特性的差异化资源分配策略，在保障用户公平性的同时，提升资源利用率，适应大规模用户的多样化业务需求。

### 3. 新兴技术融合（数字孪生、ISAC通感一体化、AI大语言大模型）

#### 3.1 RAN 数字孪生增强的无蜂窝组网技术

当前广域无蜂窝组网面临远端异步干扰分布复杂的问题，传统干扰测量方法存在精度低、开销大的瓶颈，而 RAN 数字孪生技术借助物理环境重构，结合射线追踪与统计信道建模，融合实测数据可有效获取网络中的 CSI [4,17]，为解决该问题提供了可行路径。基于此，研究提出了由边缘分布式单元、vCPU 及云化集中式单元（Cloud-CU）构成的数字孪生增强无蜂窝网络架构，可基于用户与 TRP 的空间几何位置及信道孪生建模，实现用户 - TRP 关联、配对、调度及资源分配 [10,18]，并通过协作收发机设计与异步干扰重构的迭代优化，形成网络级无线资源分配策略 [18]。但当前数字孪生建模的精确性依赖实体基站的实测数据修正，远端干扰信道测量的参考信号开销大，且协作预编码与远端干扰、多用户配对等因素高度耦合，导致网络整体性能优化的复杂度高 [18]。

未来需结合 AI 技术系统性研究基于信道数字孪生的建模方法，通过深度学习提升建模精度与实时性，降低对实测数据的依赖，减少参考信号开销 [18]；同时研究数字孪生模型的轻量化实现方案，推动其在边缘分布式单元的部署，提升实时优化能力 [10]；此外，需探索协作预编码与远端干扰抑制的联合优化算法，解耦多因素耦合关系，降低优化复杂度 [18]；还可将 RAN 数字孪生与网络管理相结合，实现无蜂窝网络的自感知、自优化，提升广域组网下的干扰抑制能力与系统整体性能 [4]。

#### 3.2 无蜂窝通感一体化（ISAC）技术

无蜂窝系统的大规模 TRP 协作能力为多站协作感知提供了基础，相较于单站感知，多站协作感知可实现目标完整视图重构并提升感知精度 [20]。当前研究将无蜂窝与通感一体化融合，提出了信号级与数据级两种感知信息融合方法，信号级融合精度高但前传负载大，数据级融合开销低但存在精度损失 [21]；同时发现 mTRP 相干 CJT 的互易性校准机制可有效解决 mTRP 协作感知的时频偏问题，提升感知时延与多普勒估计精度 [22]。但无蜂窝通感一体化仍面临诸多挑战，包括 TRP 选择中通信与感知需求的冲突、多维交叉链路干扰的管理、感知与通信的资源协同等 [23,24]，而感知与 RAN 数字孪生的融合，可实现环境信道重构，为通信性能优化提供支撑 [4]。

未来需研究信号级与数据级结合的混合感知信息融合方案，根据网络负载、目标密度及信道条件，动态调整感知处理任务在 TRP 与 vCPU 间的分配，平衡融合效率与前传负载 [21]；同时优化基于 CJT 校准机制的全网快速校准方案，设计高效的相位跟踪机制，降低校准开销并抑制相位抖动的影响 [22]；此外，需引入图论、博弈论或多目标优化方法，解决 TRP 选择中通信与感知的需求冲突，实现帕累托最优的拓扑配置 [23]；还需系统研究通感一体化场景下的干扰表征、测量与协同抑制方案，设计空间域波束零陷与多域资源协调策略，保障通信与感知性能 [24]；同时深入探索感知与 RAN 数字孪生的深度融合路径，利用环境重构实现无导频波束赋形、终端移动轨迹预测等，提升通信系统性能 [4]。

#### 3.3 AI 技术的全流程赋能

当前 AI 技术已逐步应用于无蜂窝大规模 MIMO 的部分环节，NVIDIA 提出的 AI 原生无线通信架构为 RAN 数字孪生与无蜂窝组网的融合提供了技术支撑，提升了信道建模与资源优化的精度和效率 [17]；在移动性管理方面，AI 技术的引入可实现终端移动轨迹预测，优化 TRP 集合动态调整策略，提升系统对移动性场景的支撑能力 [10]；同时，在信道估计领域，深度学习算法可实现 CSI 的精准估计与预测，减少参考信号开销 [18]。但整体而言，AI 技术在无蜂窝系统中的应用仍处于初级阶段，传统算法在复杂场景下的适应性差、复杂度高的问题尚未得到根本解决，AI 技术尚未实现对无蜂窝系统信道估计、资源调度、干扰抑制、移动性管理等全流程的深度赋能 [10,17,18]。

未来需将 AI 技术全面融入无蜂窝系统的核心环节，在信道估计层面，研发基于深度学习的高精度 CSI 估计与预测模型，提升信道信息获取的实时性与准确性，降低对参考信号的依赖 [18]；在资源调度层面，利用强化学习、深度强化学习算法，实现 TRP - 用户动态关联、时频资源分配的实时优化，适配复杂多变的网络场景 [10]；在干扰抑制层面，通过神经网络构建自适应干扰抑制模型，实现异步干扰、通感交叉干扰等复杂干扰的精准抑制 [18,24]；在移动性管理层面，优化 AI 预测算法，提升终端移动轨迹的预测精度，结合 LTM 机制进一步降低移动性管理时延 [10]；此外，需研究 AI 模型的轻量化与分布式部署方案，推动 AI 技术在边缘节点的落地，实现无蜂窝系统的自感知、自优化、自适配。

### 4. 工程落地实践（基于毫米波或太赫兹的预研验证、产业化适配与标准制定）

#### 4.1 毫米波无蜂窝大规模 MIMO 试验验证

将无蜂窝技术扩展至毫米波频段是实现高性能毫米波组网的关键路径 [19]，当前已构建完成 26 GHz 频段、200 MHz 带宽的毫米波相干协作传输试验系统，该系统包含 4 个毫米波 TRP 与 2 个间距约 300 m 的终端，基站与终端 AAU 配置 16 单元相控阵，通过 IEEE 1588 PTPv2 与 SyncE 实现同步 [25]。实测结果表明，该系统通过超分辨率估计可实现皮秒量级的通道时延差估计，基于终端辅助的校准机制可实现约 10 dB 的 CJT 波束赋形增益（理论增益 12 dB）[25]，验证了毫米波无蜂窝大规模 MIMO 相干联合传输在实际系统中的可行性。但该试验系统仍存在局限，在室内近场场景下，当用户位于波束旁瓣时，仅对数字通道校准会引入误差，导致相干联合传输的空分复用增益显著受损 [25]，且当前试验仅实现 4 个 TRP 的协作，大规模 TRP 协作下的时频同步与干扰抑制能力尚未得到验证。

未来需针对室内近场场景下的波束旁瓣校准误差问题，研究数字通道与模拟通道联合校准方案，提升空分复用增益 [25]；同时扩大试验系统的 TRP 协作规模，从 4 个 TRP 向更大规模扩展，验证大规模 TRP 协作下的时频同步、干扰抑制及资源调度能力 [19]；此外，需针对密集城区、室内办公等毫米波典型应用场景，优化 TRP 部署策略与协作传输方案，提升系统的场景适配性 [19]；还可结合实际组网需求，测试毫米波无蜂窝系统在高速移动、非视距等复杂场景下的性能，为工程落地提供实测数据支撑。

#### 4.2 太赫兹无蜂窝技术的预研与探索

太赫兹通信是 6G 频谱拓展的重要方向，当前业界关注的太赫兹频段集中在 300 GHz 以下，毫米波无蜂窝系统中的校准、协作、同步等关键技术可延续应用于太赫兹系统 [19]。与毫米波类似，太赫兹组网若采用集中式大规模阵列，会面临边缘用户速率低、易受遮挡、业务速率分布不均的问题，而传统蜂窝组网需密集部署小蜂窝，导致小区间干扰严重、波束与小区切换频繁 [19]。相控阵技术仍是太赫兹通信的主要技术途径，但受当前太赫兹相控阵芯片发展水平限制，尚未出现成熟的太赫兹多用户 RAN 试验系统 [19]，而分布式波前整形技术为太赫兹近场无线网络的分布式协作提供了技术支撑 [27]。

未来需将毫米波无蜂窝的核心技术适配至太赫兹频段，针对太赫兹载波频率更高、路径损耗更大的特点，优化 TRP 协作传输方案，提升相干协作传输的可行性 [19]；同时结合太赫兹相控阵芯片技术的发展，研发适配无蜂窝协作的太赫兹 TRP 硬件架构，实现轻量化、模块化部署 [19]；此外，深入研究太赫兹频段的分布式波前整形技术，提升近场场景下的传输性能 [27]；随着器件技术成熟，逐步构建太赫兹多用户 RAN 试验系统，验证无蜂窝分布式协作传输在太赫兹频段的性能 [19]；还需研究太赫兹无蜂窝系统的抗遮挡方案，结合多 TRP 协作提升链路的鲁棒性。

#### 4.3 产业化适配与标准制定

在标准层面，5G R19 版本已初步构建了去蜂窝化协作传输的技术框架，弱化了小区概念，支持单个 gNB 配置多个 TRP，且通过 TCI 机制实现了多小区 TRP 为同一终端服务，L1/L2 触发的移动性机制也降低了切换时延 [1,10]，而在 3GPP TSG-RAN WG1 第 123 次会议中，多家企业均提出 6GR 需从 mTRP 向无蜂窝技术演进 [3-7]，为无蜂窝技术融入 6G 标准奠定了基础。在产业化方面，已有研究基于商用 RRU 开展无蜂窝大规模 MIMO 系统试验，验证了空口互易性校准与相位同步的可行性 [12]，而全频谱无蜂窝 RAN 架构的设计也为硬件产业化提供了参考 [10]。但当前无蜂窝技术的标准化仍处于初期阶段，物理层、MAC 层、组网层的协议定义尚未完善，且适配无蜂窝分布式协作的轻量化硬件成本较高，缺乏标准化的性能测试指标与方法 [1,10,12]。

未来需推动无蜂窝技术融入 6G 3GPP 标准化体系，在 5G R19 mTRP 技术框架基础上，完善无蜂窝大规模 MIMO 的物理层传输、信道测量、反馈机制等协议定义 [1,3-7]；同时统一无蜂窝与频谱融合、SBFD、通感一体化等技术的融合协议规范，实现技术间的协同适配 [4,6,14]；在硬件产业化方面，研发适配分布式协作的轻量化、模块化 TRP 与基带处理单元，优化硬件设计并降低生产成本 [12]；此外，建立无蜂窝大规模 MIMO 的标准化性能测试体系，制定时频同步、协作传输、干扰抑制、通感一体等核心能力的测试指标与方法 [10,25]；还需研究无蜂窝技术与现有 5G/5G-A 系统的平滑演进方案，降低运营商的网络升级成本，推动产业化落地。

#### 1.1 实验路径：

##### 1.1.1 基础场景搭建与数据准备

基于 MATLAB 通信工具箱或 NS-3，构建小规模无蜂窝网络仿真场景（含 5-10 个 TRP、20-50 个用户），配置符合 3GPP R19 协议的信道模型（如 Rayleigh 信道）、子载波间隔（30kHz）及循环前缀参数

生成信道状态信息（CSI）数据集：通过仿真获取不同用户位置、移动速度下的 CSI 序列，结合历史 SRS/DMRS 数据构建训练集（含信道时延、频偏、相位信息）

##### 1.1.2 核心算法设计与实现

信道估计 / 预测模块：基于 Python 实现 LSTM/GRU 模型，输入历史 CSI 序列，输出未来 1-5 个时隙的 CSI 预测结果；对比传统最小二乘（LS）估计算法，验证模型在信道跟踪精度上的优势

资源调度模块：基于强化学习（Q-Learning/DQN）设计 TRP - 用户动态关联与时频资源分配算法，以频谱效率最大化、调度时延最小化为目标，构建奖励函数；在 NS-3 的 D-MAC 架构中嵌入算法，模拟多 TRP 协作调度过程

##### 1.1.3 非理想场景适配与优化

引入非理想回传约束（设置 10-50ms 回传时延、5%-10% 丢包率），优化算法的鲁棒性（如加入时延补偿机制、轻量化模型降低计算开销）

扩展至大规模用户场景（100 + 用户），验证算法的可扩展性，确保调度复杂度呈线性增长

##### 1.1.4 性能评估与对比分析

核心指标：CSI 预测误差（NMSE）、频谱效率（bps/Hz）、调度时延、用户公平性（ Jain 指数）

对比组：传统贪心调度算法、LS 信道估计算法、无 AI 优化的分布式调度方案

结果呈现：通过折线图展示不同场景下的指标对比，分析 AI 算法的性能提升幅度

#### 1.2 创新点：

**轻量化模型设计**：针对无蜂窝终端算力有限的问题，简化 LSTM/GRU 模型结构（减少隐藏层神经元数量），结合剪枝技术降低模型复杂度，在保证预测精度的同时提升运行效率

**多目标强化学习调度**：突破单一目标调度局限，在奖励函数中融入频谱效率、时延、能耗多维度指标，实现 TRP - 用户关联与资源分配的协同优化，提升复杂场景下的用户体验

**非理想回传自适应机制**：设计基于链路状态感知的算法参数动态调整策略，根据回传时延 / 丢包率自适应切换调度周期与 CSI 更新频率，解决非理想场景下的性能下降问题

**CSI 与业务特性融合**：在信道预测中引入用户业务类型（如大上行 / 大下行业务）权重，优先保障高优先级业务的信道估计精度，提升资源分配的针对性

### 二、无蜂窝通感一体化的信息融合与干扰抑制算法

#### 2.1 实验路径：

##### 2.1.1 通感一体化场景建模

基于 MATLAB 构建无蜂窝 ISAC 仿真平台，配置 6-8 个分布式 TRP（支持通信与感知双模）、3-5 个目标用户 / 感知目标，通信频段选用中频段（FR3），感知目标为静止 / 低速移动物体（如室内人员、设备）

定义通信与感知参数：通信采用 OFDM 调制（带宽 20MHz），感知采用脉冲多普勒体制，设置通感频谱共享模式（时分 / 频分复用）

##### 2.1.2 信息融合算法实现与验证

数据级融合模块：在 TRP 端提取感知目标的多普勒 - 距离特征、通信信号的信道增益特征，基于 Python 实现图论加权融合算法（如 K 近邻加权），将多 TRP 特征融合为统一感知结果

融合性能评估：对比单 TRP 感知与多 TRP 融合的定位误差（RMSE）、目标识别准确率，验证融合算法的优势

##### 2.1.3 干扰抑制策略设计与测试

构建多维干扰模型：模拟通信 - 感知交叉链路干扰（CLI）、TRP 间感知干扰，基于 MATLAB 实现空间域波束零陷算法（在通信波束中对感知目标方向设置零陷）

资源协调优化：设计通感时频资源分配策略，为通信高优先级用户分配独立频谱块，避免与感知信号冲突

##### 2.1.4 系统性能综合评估

核心指标：通信频谱效率、误码率（BER）、感知定位误差、干扰抑制比（SIR）

对比组：无干扰抑制的通感共享方案、单 TRP 通感一体化方案

结果呈现：通过柱状图展示干扰抑制前后的性能变化，分析融合算法与干扰策略的协同效果

#### 2.2 创新点：

**低成本数据级融合方案**：针对前传负载约束，提出 “特征筛选 + 轻量化加权” 融合算法，在 TRP 端剔除冗余感知特征，仅传输关键信息，降低前传开销的同时保证定位精度

**通感协同的 TRP 选择机制**：引入多目标优化（通信信道增益最大化 + 感知几何角度最优），基于博弈论设计 TRP - 用户关联策略，解决通信与感知的 TRP 选择冲突

**自适应波束零陷干扰抑制**：结合感知目标位置动态调整通信波束零陷方向，避免固定零陷导致的通信性能损失，实现干扰抑制与通信覆盖的平衡

**通感资源智能分配**：基于感知目标密度与通信业务量动态切换通感复用模式（时分 / 频分），在感知目标密集时采用频分复用减少干扰，在通信业务繁忙时采用时分复用提升频谱效率
