# Python-MATLAB 混合架构 QA 总结

## Q1：为什么需要 Python + MATLAB？

MATLAB 负责成熟的通信仿真链路，Python 负责 GNN 训练与推理。这样可以保留 MATLAB 在矩阵运算、信道建模、预编码和论文绘图上的优势，同时使用 PyTorch/PyG 实现图神经网络。

## Q2：两边如何交换数据？

训练阶段由 MATLAB 导出 `.mat` 数据，Python 读取后训练模型；仿真阶段 MATLAB 调用 Python 推理模块，得到功率矩阵 `rho` 后继续计算 SE/ESR。

```mermaid
flowchart LR
    A["MATLAB 信道与 WMMSE"] --> B["训练数据 .mat"]
    B --> C["Python GNN 训练"]
    C --> D["best_gat_gnn_power.pt"]
    D --> E["MATLAB 调 Python 推理"]
    E --> F["ESR 评估和绘图"]
```

## Q3：MATLAB 不能直接训练 GNN 吗？

理论上可以，但需要自己维护图结构、消息传递、批处理、自动求导细节和模型保存协议。PyTorch Geometric 已经覆盖这些基础设施，更适合当前 GNN 训练任务。

## Q4：MATLAB 调 Python 有哪些注意点？

1. `pyenv` 必须指向装有 `torch`、`torch_geometric`、`numpy` 的 Python 环境。
2. MATLAB 进程中 Python 解释器一旦加载，路径很难动态切换，建议启动 MATLAB 前确认环境。
3. `computeRhoGNN.m` 与 `python/inference/__init__.py` 的特征 schema 必须和 `python/train_gnn.py` 保持一致。

## Q5：当前架构的分工原则是什么？

| 模块 | 实现侧 | 原因 |
|---|---|---|
| 信道生成、预编码、ESR | MATLAB | 项目已有实现稳定，矩阵计算方便 |
| WMMSE 标签生成 | MATLAB | 与主仿真参数保持一致 |
| GNN 数据集、训练、推理 | Python | PyTorch/PyG 生态成熟 |
| 绘图与缓存 | MATLAB | 直接复用主仿真结果结构 |

## Q6：如果后续扩展新学习方法怎么办？

优先保证三件事：训练数据字段清晰、推理端特征构造和训练端完全一致、主仿真的算法表和绘图逻辑按语义标签匹配，而不是按固定行号匹配。
